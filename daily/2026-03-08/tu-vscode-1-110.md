# VS Code 1.110 — エージェント機能の本格化

- **日付**: 2026-03-08
- **カテゴリ**: Dev Tool
- **ソース**: [VS Code Updates](https://code.visualstudio.com/updates/v1_110), [Visual Studio Magazine](https://visualstudiomagazine.com/articles/2026/03/04/vs-code-1-110-ships-with-agent-plugins-browser-tools-and-session-memory.aspx), [GitHub Changelog](https://github.blog/changelog/2026-03-06-github-copilot-in-visual-studio-code-v1-110-february-release/)

## 概要

VS Code 1.110（February 2026 リリース、2026年3月4日公開）は、AI エージェントをより長期的で複雑なタスクに実用的に使えるようにすることにフォーカスしたリリース。エージェントプラグイン、ブラウザ自律操作、セッションメモリ、コンテキストコンパクションなど、エージェント機能が大幅に強化された。

## 主な変更点

### エージェントプラグイン（実験的）

スキル・コマンド・エージェント・MCP サーバーをバンドルした「エージェントプラグイン」を拡張機能ビューから直接インストール可能に。`@agentPlugins` で検索でき、GitHub/Git リポジトリを通じて配布される。これにより、エージェントの能力を拡張するエコシステムが形成される。

### エージェンティックブラウザツール（実験的）

統合ブラウザをエージェントが自律的に操作できるようになった。主な機能:

- **ページナビゲーション**: URL を指定してブラウザを開く
- **コンテンツ読み込み**: ページ内容の取得・解析
- **スクリーンショット**: ページの視覚的な状態をキャプチャ
- **ユーザー操作のシミュレーション**: クリック、テキスト入力など
- **Playwright 統合**: カスタムブラウザ自動化スクリプトの実行

エージェントが自分の変更結果を Web ブラウザで検証できるため、開発ループが閉じる。

### セッションメモリ

Plan エージェントが作成した計画が会話全体を通じて保持されるようになった。コンテキスト圧縮後も計画は利用可能で、長時間の作業セッションでの一貫性が向上。

### コンテキストコンパクション

会話履歴を自動的に要約してコンテキストウィンドウを効率的に利用。`/compact` コマンドで手動トリガーも可能。ローカル・バックグラウンド・Claude エージェント全てで利用できる。

### セッションフォーク

会話履歴を継承した独立セッションを作成可能。`/fork` コマンドで全履歴を継承、またはホバーメニューから任意の地点でフォークできる。異なるアプローチを並行して試す際に有用。

### エージェントデバッグパネル（プレビュー）

チャットイベント、システムプロンプト、ツール呼び出しをリアルタイムで可視化。エージェントの動作を理解・デバッグする際に活用できる。

### チャットからのカスタマイズ生成

会話中に直接プロンプト・スキル・エージェントを作成可能:
- `/create-prompt`: プロンプトの作成
- `/create-skill`: スキルの作成
- `/create-agent`: エージェントの作成

### その他の改善

- **Kitty グラフィックスプロトコル対応**: PNG/RGB 画像をターミナルで直接レンダリング
- **Claude 互換性**: Claude の設定ファイル（`CLAUDE.md` 等）を VS Code で直接再利用可能
- **アクセシビリティ改善**: スクリーンリーダーサポート、キーボードナビゲーション、チャットインタラクションの認識が向上

## 今後の注目点

- **エージェントプラグインエコシステム**: サードパーティ製プラグインの登場により、エージェントの能力がどこまで拡張されるか
- **ブラウザツールの安定化**: 現在は実験的機能だが、E2E テストやビジュアルリグレッションテストとの統合が期待される
- **メッセージステアリング・キューイング**: リクエスト処理中にフォローアップメッセージを送信できる機能により、エージェントとの対話がより自然に

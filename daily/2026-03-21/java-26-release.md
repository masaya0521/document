# Java 26リリース — 言語進化とAI統合の新展開

- **日付**: 2026-03-21
- **カテゴリ**: Tech
- **ソース**: [Oracle](https://www.oracle.com/news/announcement/oracle-releases-java-26-2026-03-17/), [InfoQ](https://www.infoq.com/news/2026/03/java26-released/), [InfoWorld](https://www.infoworld.com/article/4050993/jdk-26-the-new-features-in-java-26.html)

## 概要

Oracleは2026年3月17日にJava 26（JDK 26）をリリースした。JDK 25以来初の非LTSリリースで、10個のJEP（JDK Enhancement Proposals）を含む。言語のパターンマッチング強化、HTTP/3サポート、GCスループット改善、暗号化機能の充実など、開発者の生産性向上とAI・セキュリティ統合を推進する内容となっている。

## 詳細

### 言語機能

#### パターンにおけるプリミティブ型（JEP 530）— 第4プレビュー
JDK 23から続くプレビュー機能。パターンマッチングをすべてのプリミティブ型に拡張し、`instanceof`や`switch`でプリミティブ値のマッチング・テスト・安全な変換が可能になる。これにより既存のパターンマッチングの制限が解消される。

### パフォーマンス・ランタイム

#### AOTオブジェクトキャッシュ（JEP 516）
事前キャッシュをすべてのGCで利用可能にする機能強化。低遅延ZGCを含む任意のGCで、GC非依存の中立フォーマットからキャッシュされたJavaオブジェクトをメモリに順次ロードできるようになった。

#### G1 GCスループット改善（JEP 522）
G1ガベージコレクタ使用時のアプリケーションスループットを向上。アプリケーションスレッドとGCスレッド間の同期量を削減することで実現。

### ライブラリ

#### HTTP/3サポート（JEP 517）
HTTP Client APIをHTTP/3プロトコルに対応させ、最小限のコード変更でHTTP/3サーバーとの通信が可能に。QUICベースの次世代プロトコルへの移行を支援する重要な追加。

#### Vector API（JEP 529）— 第11インキュベーション
ベクトル計算を表現するAPIで、サポートされるCPUアーキテクチャ上で最適なベクトル命令にコンパイルされる。JDK 25からの実質的な実装変更はなし。

#### Lazy Constants（JEP 527）
旧「Stable Values」から改名。遅延初期化される定数値を扱うための高レベルAPIで、名前変更により使用意図がより明確になった。

#### 構造化並行性（JEP 480）— 第6プレビュー
マルチスレッドプログラミングを簡素化する機能。タスクのグループ化とエラー処理を構造的に管理できる。

#### PEMエンコーディングサポート（JEP 523）
暗号オブジェクトをPEM（Privacy-Enhanced Mail）形式でエンコード・デコードするためのプレビューAPI。

### セキュリティ・クリーンアップ

#### Applet API削除（JEP 504）
JDK 17で削除予定とされていたApplet APIを正式に削除。ブラウザプラグインの廃止に伴い、長年の技術的負債を清算。

#### Deep Reflectionに関する警告（JEP 531）
finalフィールドの変異にディープリフレクションを使用する安全でない操作について警告を発する機能。将来的なアクセス制限への移行を段階的に進める。

### その他の注目機能

- **ハイブリッド公開鍵暗号化（HPKE）**: 業界標準の暗号化を簡素化
- **ポスト量子対応JAR署名**: サプライチェーンの将来性を確保
- **Unicode 17.0 / CLDR v48**: 国際標準のサポート更新

## 今後の注目点

- Structured Concurrency（構造化並行性）が正式機能に昇格するタイミング — 6回目のプレビューとなり成熟度は高い
- Vector APIの11回目のインキュベーションが示す慎重なアプローチと、Project Valhalla（値型）との統合時期
- HTTP/3サポートの実運用でのパフォーマンス評価
- JDK 27（2025年9月予定の次期LTS候補）に向けた機能の安定化状況
- Primitive Types in Patternsの最終化時期 — 4回目のプレビューで安定に近い段階

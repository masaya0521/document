# 情報セキュリティ10大脅威2026 — 「AI利用をめぐるリスク」初ランクインの意味

- **日付**: 2026-06-11
- **カテゴリ**: Japan Tech
- **ソース**: [IPA 情報セキュリティ10大脅威 2026](https://www.ipa.go.jp/security/10threats/10threats2026.html)

## 概要

IPA（情報処理推進機構）が公表した「情報セキュリティ10大脅威 2026」の組織向けランキングで、「AIの利用をめぐるサイバーリスク」が**初選出で第3位**にランクインした。生成AIの業務利用が一気に広がった結果、AIそのものが攻撃対象・攻撃手段の両面でリスク要因として認識される段階に入ったことを示す。1位はランサム攻撃、2位はサプライチェーン・委託先を狙った攻撃で、上位2つは引き続き深刻な脅威として残った。

## 詳細

### 組織向けトップ10（2026）

| 順位 | 脅威 |
|------|------|
| 1 | ランサム攻撃による被害 |
| 2 | サプライチェーンや委託先を狙った攻撃 |
| 3 | **AIの利用をめぐるサイバーリスク（初選出）** |
| 4 | システムの脆弱性を悪用した攻撃 |
| 5 | 機密情報を狙った標的型攻撃 |
| 6 | 地政学的リスクに起因するサイバー攻撃 |
| 7 | 内部不正による情報漏えい等 |
| 8 | リモートワーク等の環境や仕組みを狙った攻撃 |
| 9 | DDoS攻撃 |
| 10 | ビジネスメール詐欺 |

### 「AIの利用をめぐるサイバーリスク」とは何か

このカテゴリは単一の攻撃手法ではなく、AIの普及に伴う複合的なリスク群を指す。具体的には次のような側面が含まれる。

- **攻撃の高度化**: AIによって、より自然な日本語のフィッシングメールが大量生成される。脆弱性を突く攻撃コードの生成もAIで効率化され、攻撃の量・質ともに底上げされる。
- **AIシステム自体への攻撃**: プロンプトインジェクション、学習データ汚染、機密情報を引き出すような入力など、業務に組み込んだAIが新たな攻撃面（アタックサーフェス）になる。
- **情報漏えいリスク**: 従業員が生成AIに機密情報を入力してしまう、AIの出力経由で内部情報が外部に流出するといった、運用上の漏えい経路。

「試す段階」から「社会実装する段階」へAIが移行した2026年において、利便性の裏側にあるリスクが顕在化したことが、初ランクイン・いきなり3位という結果に表れている。

### 実際のインシデントとの符合

6月には、AIを活用したフィッシングや攻撃コード生成が現実の脅威として報じられ、興亜硝子や複数企業・自治体での不正アクセスも相次いだ。ランキングが示す傾向と、足元のインシデント発生状況は一致している。

## 今後の注目点

- 企業の生成AI利用ポリシー整備（入力禁止情報の明確化、ログ監査）がどこまで進むか
- プロンプトインジェクション等、AI固有の脆弱性に対する防御技術・ガイドラインの標準化
- 攻撃側のAI活用に対し、防御側がAIをどう活用して非対称性を埋めるか
- サプライチェーン攻撃（2位）とAIリスク（3位）が組み合わさる複合脅威への備え

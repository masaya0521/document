# GPT-5.4 — 100 万トークン × 自律ワークフローが意味するもの

- **日付**: 2026-05-19
- **カテゴリ**: Tech
- **ソース**: [DataCamp](https://www.datacamp.com/blog/gpt-5-4) / [The AI Insider](https://theaiinsider.tech/2026/03/06/openai-launches-gpt-5-4-with-expanded-context-window-improved-reasoning-and-higher-benchmark-performance/) / [Awesome Agents](https://awesomeagents.ai/news/gpt-5-4-computer-use-office-launch/) / [llm-stats.com](https://llm-stats.com/ai-news)

## 概要

OpenAI が発表した GPT-5.4 は「professional work 向けの最も能力が高く効率の良いフロンティアモデル」と位置付けられ、 (1) 約 105 万トークンの context window、(2) ネイティブな computer use (画面・キーボード・マウス操作)、(3) フル解像度ビジョン、を単一モデルに統合した。OSWorld-Verified ベンチマークで 75.0% を記録し、人間ベースライン 72.4% を上回ったうえ、GPT-5.2 (47.3%) からは 1 世代で約 27pt の跳躍となる。チャットアシスタントから「自律的なデジタル同僚」への移行を象徴するモデルである。

## 詳細

### 1M トークン context が変えるエージェント設計

これまでのエージェント設計は、長期タスクを RAG・要約・外部メモリで分割することで context 制限を回避してきた。GPT-5.4 は 1,050,000 トークン (約 75 万語) を素のコンテキストで保持できるため、以下のような新しい設計余地が生まれる。

- **長尺タスクの「先読みと振り返り」を 1 セッションで完結**: 過去の操作ログ・コード差分・ドキュメントを全て同梱したまま意思決定でき、ステップ間でエージェントが「直前に何をしたか」を忘れる典型的な失敗が減る。
- **コードベース全体を context 内に投入できる規模**: 中規模リポジトリなら丸ごと入る。Codex 内蔵で IDE エージェントの「リポジトリ理解」精度が一段上がる。
- **ツール呼び出しチェーンの履歴を圧縮せず保持**: 数十回のツール呼び出しと観測結果を要約せずに残せるため、エージェント挙動の説明性・デバッグ性が向上。

### Computer Use — モデル側に「PC を操作する能力」が組み込まれた

GPT-5.4 は OpenAI のメインライン (Pro / Thinking ではなく標準のフラッグシップ) では初めて、computer use を「事後接続したツール」ではなく「学習段階から組み込まれた能力」として持つ。OSWorld-Verified はスクリーンショット + 入力デバイスのみで構成された実際のデスクトップタスク (ファイル整理・ブラウザ操作・スプレッドシート編集など) を評価するベンチマークで、ここで人間を上回ったことは「画面を見て自分で操作するエージェント」が現場運用フェーズに入ったことを意味する。

### バリアント構成と用途

- **Standard**: 既定の API 露出。
- **Pro**: 高負荷ワークロード向けに最適化。
- **Thinking**: 推論時に内部熟考を強化、長期計画タスク向け。

API と Codex から直接利用可能。価格・レートリミットは「professional work」向けに振った設計で、純粋なチャット用途の汎用 LLM 競争とはセグメントが分かれつつある。

## 今後の注目点

- **エージェントの実害域の拡大**: 自律的に PC を操作する能力が一般化すると、誤操作・データ破壊・横展開ミスのリスクが現実化する。サンドボックス境界、操作ログ、承認ゲートをどう標準化するかが急務。
- **Anthropic / Google の対抗策**: Anthropic は Claude の computer use を先行投入していた領域。Google I/O 2026 では Gemini Intelligence + Android XR で「OS とデバイス側からの統合」を提示する見込みで、エージェント市場が「モデル単体勝負」から「プラットフォーム勝負」に進む。
- **コンテキスト課金モデルの変化**: 1M tokens を毎回ロードするとコストが急増する。プロンプトキャッシュやデルタ入力課金など、サブスクリプション側のメカニズム再設計が進むはず。
- **既存 RAG/エージェントフレームワークの再評価**: 「短い context を前提に組まれたチェーン」が無用な複雑さになる可能性。LangChain / LlamaIndex 等のレイヤーがどう適応するか。
- **エンタープライズ採用**: JPMorgan が AI を「コアインフラ」に位置付け 198 億ドルを投資する流れに、GPT-5.4 の自律実行能力がどこまで噛み合うか。

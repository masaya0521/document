# Next.js — 2026 年 5 月コーディネートセキュリティリリース

- **日付**: 2026-05-08
- **カテゴリ**: Frontend
- **ソース**: [Vercel changelog](https://vercel.com/changelog/next-js-may-2026-security-release), [Netlify まとめ](https://www.netlify.com/changelog/2026-05-08-react-nextjs-security-vulnerabilities/)

## 概要

Vercel が Next.js に対して **13 件のアドバイザリを束ねたコーディネートセキュリティリリース** を実施した。修正カテゴリは認可バイパス、サービス妨害（DoS）、サーバーサイドリクエストフォージェリ（SSRF）、キャッシュポイズニング、クロスサイトスクリプティング（XSS）と多岐にわたる。リリース時点で「アップグレード以外の完全な緩和策はない」と明言されており、影響範囲が広いため即時のパッチ適用が推奨される。

## 修正バージョン

| 系列 | 修正バージョン |
|------|----------------|
| Next.js 15.x | **15.5.18** |
| Next.js 16.x | **16.2.6** |
| react-server-dom-* | 19.0.6 / 19.1.7 / 19.2.6 |

Next.js 13.x / 14.x は全バージョンが対象。15.x / 16.x は上記未満のバージョンが影響を受ける。

## 主な変更点

### 認可バイパス系（5 件）

- **App Router セグメントプリフェッチでの認可バイパス**（高リスク 2 件）。プリフェッチ経由でアクセスすべきでないルートが返ってしまうケースを修正
- **Pages Router の i18n 機能経由での代理認可回避**
- **動的ルートパラメータの注入**による認可ロジック迂回
- **ミドルウェアリダイレクトのキャッシュポイズニング**

### サービス妨害（3 件）

- **CVE-2026-23870**: React Server Components の処理で過大なリソース消費を誘発できる DoS（React 上流の問題、Server Components を利用するアプリ全体に波及）
- **Cache Components 利用時の接続枯渇 DoS**
- **画像最適化 API 経由の攻撃**による DoS

### SSRF / キャッシュポイズニング / XSS

- **SSRF**: WebSocket アップグレードを使うアプリでサーバ起点リクエストを誘発可能
- **キャッシュポイズニング**: React Server Component レスポンスのキャッシュへの汚染、および RSC のキャッシュバスティング衝突を悪用したケースの 2 件
- **XSS**: App Router で CSP nonce / `beforeInteractive` スクリプトに未信頼入力が混入した際の 2 件（深刻度は moderate / low）

## 破壊的変更・移行ガイド

API 仕様変更はないため、パッチバージョンアップで完了する。

- 15.x 系: `15.5.18` 以上へ
- 16.x 系: `16.2.6` 以上へ
- React Server Components を直接利用しているケースでは `react-server-dom-*` も対応版へ揃える

WAF ルールでの迂回は不可能と明言されており、**パッチ適用が唯一の完全な緩和策**。

## 今後の注目点

- Cache Components / RSC のキャッシュ周りはここ数ヶ月で複数の重大バグが報告されており、デプロイ前にキャッシュキー設計を再点検する価値がある
- App Router のセグメントプリフェッチは「クライアントから自由に叩ける副作用つきエンドポイント」になりがちで、認可は **route handler 側でも必ず再確認** する設計が安全
- 13.x / 14.x はもう投影外で常に「全バージョン影響」と扱われる。LTS 的に使い続けている場合は 15.x 以降への移行計画を立てる時期

# Go — 1.25.10 / 1.26.3 セキュリティリリース

- **日付**: 2026-05-07（一部資料は 2026-05-08 を記載）
- **カテゴリ**: Language
- **ソース**: [golang-announce アナウンス](https://groups.google.com/g/golang-announce/c/qcCIEXso47M), [Release History](https://go.dev/doc/devel/release)

## 概要

Go 1.25.10 と Go 1.26.3 が同時にリリースされた。標準ライブラリと `go` コマンドを横断する **11 件のセキュリティ修正** に加え、コンパイラ・リンカ・ランタイムのバグ修正も含む。`html/template`、`net`、`net/http`、`net/http/httputil`、`net/mail`、`syscall`、`crypto/fips140`、`go/types`、`os` などほぼ全領域に手が入る大型パッチで、サプライチェーン保護に直結する **モジュールチェックサム検証バイパス** が目玉。

## 主な変更点（CVE 一覧）

| CVE | 影響パッケージ | 内容 |
|-----|----------------|------|
| CVE-2026-42501 | `cmd/go` | 悪意あるモジュールプロキシが **モジュールチェックサム DB の検証をバイパス可能**。サプライチェーン攻撃のリスク大 |
| CVE-2026-39819 | `cmd/go` | `go bug` が予測可能な一時ファイル名を使用しシンボリックリンク攻撃に脆弱 |
| CVE-2026-39817 | `cmd/go` | `go tool pack` が抽出時にファイル名をサニタイズせず、悪意あるアーカイブで任意ファイルを書き込み可能。今後は **ディレクトリ成分を含むエントリを拒否** |
| CVE-2026-39825 | `net/http/httputil` | `ReverseProxy` がクエリパラメータ上限を無視して転送 |
| CVE-2026-33814 | `net/http` | HTTP/2 で `SETTINGS_MAX_FRAME_SIZE = 0` を送られると無限ループ |
| CVE-2026-39836 | `net` | Windows で `Dial` / `LookupPort` に NUL バイトを含む入力でパニック。今後は **エラーを返す** ように修正 |
| CVE-2026-33811 | `net` | cgo DNS リゾルバで長い CNAME 応答時にメモリの二重解放 |
| CVE-2026-42499 | `net/mail` | `consumePhrase` の二次計算量に起因する DoS |
| CVE-2026-39820 | `net/mail` | `consumeComment` で過度な CPU 消費による DoS |
| CVE-2026-39826 | `html/template` | `type` 属性が空 / 空白文字のみのときに XSS |
| CVE-2026-39823 | `html/template` | `meta` の content URL エスケープを回避できる XSS |

## 破壊的変更・移行ガイド

API 仕様の互換性は保たれている。ただし以下の挙動変更は意識しておくとよい。

- `net.Dial` / `net.LookupPort` は Windows で NUL バイトを受け取ると **panic ではなく error を返す**。`recover` で握りつぶしていたコードは早期に失敗するようになる
- `go tool pack` は **ディレクトリ成分を含むエントリを抽出しない**。サードパーティのビルドスクリプトでこれに依存していた場合は調整が必要

## 推奨アクション

1. https://go.dev/dl/ から該当バージョンを取得
2. CI / Dev コンテナの Go バージョンを更新
3. CVE-2026-42501（モジュールプロキシ経由の checksum バイパス）は **プライベートプロキシ運用者にも影響** するため、自前 Athens / 内製プロキシを動かしている環境は優先的に検証
4. リバースプロキシで `net/http/httputil.ReverseProxy` を使っている場合、CVE-2026-39825 と CVE-2026-33814 の組み合わせで上流に DoS が伝播する可能性があるため早めの更新を推奨

## 今後の注目点

- 過去半年で Go のセキュリティリリースは月次ペース化しつつあり、本番ビルドのバージョン固定戦略を見直す時期
- `go tool pack` のパストラバーサル修正は、過去のアーカイブ系 CVE（tar / zip 系）と同パターン。ビルドキャッシュやコンテナイメージビルドのパイプラインに「ファイル名のサニタイズ」を組み込んでおくと将来の似た脆弱性にも対応しやすい
- HTTP/2 系（CVE-2026-33814）はクライアント側を巻き込む可能性があるため、API クライアント用途で組み込んでいるサービスも更新対象に含めること

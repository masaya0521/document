# Deno — 2.8（過去最大のマイナーリリース）

- **日付**: 2026-06-24（リリースは 2026-05-22）
- **カテゴリ**: Backend / Runtime
- **ソース**: [Deno Blog v2.8](https://deno.com/blog/v2.8), [Release v2.8.0（GitHub）](https://github.com/denoland/deno/releases/tag/v2.8.0)

## 概要

2026-05-22 にリリースされた Deno 2.8 は、Deno チーム自身が「過去最大のマイナーリリース」と位置づける大型アップデート。依存パッケージのセキュリティ対応、パッケージ公開フロー、Node.js 互換性、起動・インストール性能など広範囲で改善が入った。直近の TypeScript 7.0 RC（Go ネイティブコンパイラ）や Bun 1.3 のフルスタック化と並んで、JavaScript ランタイム競争が依然として活発であることを示す内容になっている。

## 主な変更点

### パッケージ管理・セキュリティ

- 依存脆弱性チェックに **自動修正** を追加。バージョン制約の範囲内で、脆弱な npm パッケージをパッチ済みバージョンへ引き上げる。
- `deno.json` / `package.json` の **version フィールドを更新するコマンド** を追加。patch / minor / major / prerelease を指定可能。
- Deno / JSR プロジェクトを **npm 公開用 tarball に変換するコマンド** を追加。トランスパイル済み JS、抽出した `.d.ts`、specifier の自動書き換えを含む。

### パフォーマンス

- React・Vite・Babel parser・ESLint を import するエントリポイントの初回インストールが **3.66 倍高速化**（クリーンな `DENO_DIR` で 3,319ms → 906ms）。
- V8 エンジンを 14.6 → **14.9** へアップグレード。

### Node.js 互換性

- Node のテストスイート通過率が 2.7 の約 42% から **76.4%** へ大幅向上。npm エコシステムとの互換性がさらに強化された。

## 破壊的変更・移行ガイド

メジャーリリースではないため、互換性を破る変更は基本的に含まれない。`deno upgrade` で更新可能。V8 14.9 への更新に伴う挙動差が気になる場合は、CI で既存テストを実行して回帰を確認するとよい。

## 今後の注目点

- **Node 互換率の伸び**: 42% → 76.4% という急上昇は、既存 Node アプリの移行ハードルを下げる。次リリースでさらに 80% 台に乗るか。
- **依存脆弱性の自動修正**: サプライチェーン対策として、`npm audit fix` 相当の体験をランタイム標準で提供する流れ。Bun 1.3 の Scanner API とあわせ、ランタイム側のセキュリティ機能競争が焦点。
- **2.9 / 3.0 の方向性**: 「過去最大のマイナー」を出した後、次の大型機能がどこに向かうか（パフォーマンス・互換性・DX）。

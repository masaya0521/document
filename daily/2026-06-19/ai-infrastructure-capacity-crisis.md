# AI インフラの容量危機 — 計算資源の取り合いが始まった

- **日付**: 2026-06-19
- **カテゴリ**: Tech
- **ソース**: [unrot.co AI News June 17](https://unrot.co/blogs/ai-news-today-june-17-2026), [TechStartups](https://techstartups.com/2026/06/16/top-tech-news-today-june-16-2026/)

## 概要

2026 年 6 月、AI 業界の構造的なボトルネックが「モデルの賢さ」から「計算資源の確保」へと明確にシフトしたことを示す出来事が相次いだ。AI コーディングエージェントの普及で開発インフラが想定外の負荷を受け、クラウド最大手ですら社内の容量では需要を満たせず外部から計算資源を借りる事態に陥っている。需要が各社のインフラ計画サイクルを追い越したことが、業界横断的に露呈した格好だ。

## 詳細

### GitHub がエージェントの PR に押し潰される

GitHub 上で AI コーディングエージェントが生成するプルリクエストは、2025 年 9 月の月 400 万件から 2026 年 3 月には月 1700 万件へと約 4 倍に急増した。この負荷で GitHub の可用性は 88.4% まで低下し、Microsoft は自社（Azure）だけでは捌けず、競合である Amazon Web Services の容量を使ってトラフィックを迂回させる対応に追い込まれた。「Microsoft の製品が AWS の上で動く」という象徴的な事態である。

### Google ですら外部の計算資源を買う

世界最大級のクラウド事業者である Google でさえ、AI 需要を社内で賄えていない。報道では、Google が xAI の Colossus 施設の計算資源を 2029 年まで利用するため、SpaceX に月額 9.2 億ドルを支払っているとされる。クラウドのインフラを「提供する側」が、別企業からインフラを「借りる側」に回るという逆転が起きている。

### 資本集約型産業への転換

この容量危機は、SpaceX による Cursor 買収（$60B）、DeepSeek の $7.4B 調達、Qualcomm による Tenstorrent 買収交渉（$8〜10B）、Nvidia の Rubin プラットフォーム発表といった「インフラ・ハードウェアへの巨額投資」の流れと地続きだ。AI 産業の競争軸は、優れたモデルやアプリケーションだけでなく、チップ・データセンター・電力という物理的な土台の奪い合いへと移りつつある。

## 今後の注目点

- AI エージェントによる開発トラフィックの増加が、CI/CD・コードホスティング・レビュー基盤の設計にどんな影響を与えるか
- 「クラウド事業者が他社から計算資源を借りる」構図が一時的なものか、構造的なものとして定着するか
- Nvidia Rubin など次世代チップの供給が需要ギャップをどこまで埋められるか
- 計算資源の偏在が、スタートアップと大手の競争格差をさらに広げないか

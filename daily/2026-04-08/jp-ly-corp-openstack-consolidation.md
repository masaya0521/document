# LY Corporation：164 OpenStack クラスタを1つに統合する大規模インフラ刷新

- **日付**: 2026-04-08
- **カテゴリ**: Japan Tech
- **ソース**: [The Register](https://www.theregister.com/2026/04/07/ly_corp_openstack_consolidation/)

## 概要

LY Corporation（旧 LINE ヤフー）が、Yahoo! Japan と LINE の統合に伴い、合計164以上の OpenStack クラスタを単一の新クラウド基盤「Flava」に統合するプロジェクトを進めている。29万 VM・3.8万サーバーという大規模インフラの統合であり、カスタマイズ過多によるアップグレード困難と、政府指導によるセキュリティ改善が主な動機となっている。

## 詳細

### 統合前の規模

LY Corporation は2023年の Yahoo! Japan と LINE の経営統合により、2つの巨大なプライベートクラウドを抱えることになった。

**LINE 側「Verda」クラウド：**
- 4つの OpenStack クラスタ
- 130,000 VM
- 11,000 ホスト

**Yahoo! Japan 側「YNW」クラウド：**
- 160以上の OpenStack クラスタ
- 160,000以上の VM
- 27,000 サーバー

### 統合の動機

#### 技術的課題
LY社のクラウドインフラストラクチャ責任者は「レガシークラウドでは OpenStack へのカスタム修正が多すぎたため、アップグレードが困難になった」と説明している。160以上のクラスタそれぞれに独自のカスタマイズが施されており、運用・保守コストが膨大になっていた。

#### セキュリティ問題
2023年に発生したユーザーデータの漏洩事件を受け、日本政府が LY Corporation に対してセキュリティとプライバシーの改善を指示。技術スタック全体の見直しが求められた。

### 新統合クラウド「Flava」のアーキテクチャ

Flava は以下の3原則に基づいて設計されている：

1. **ステートレス性の追求** — 永続データは外部ストレージ（Ceph）に移行し、コンピュートノードをステートレスに保つ
2. **アプリケーション駆動の可用性** — インフラ側だけでなくアプリケーション設計で信頼性を確保
3. **迅速な復旧** — Infrastructure as Code（IaC）による環境の迅速な再構築を可能に

技術スタック：
- 監視：Prometheus、Grafana
- ネットワーク：Envoy、FRRouting
- ストレージ：Ceph
- 可観測性：eBPF

最も重要な方針として、**OpenStack のアップストリームに忠実であること**を掲げている。カスタムパッチを最小限に抑え、機能変更が必要な場合はアップストリームに貢献するアプローチを採る。

### 統合後の規模

初期段階では：
- 単一の OpenStack クラスタ
- 9,000以上の VM
- 500以上のホスト

全体のワークロードが段階的に Flava に移行される予定。

## 今後の注目点

- 29万 VM から9,000 VM への大幅削減が示す、コンテナ化・最適化の進捗
- アップストリーム忠実方針の実効性と OpenStack コミュニティへの貢献
- セキュリティ改善の具体的な成果と政府の評価
- 統合完了までのタイムラインとサービス影響
- LINE と Yahoo! Japan のサービス統合に伴うアーキテクチャ上の課題

# Deno — v2.9 リリース

- **日付**: 2026-06-30
- **カテゴリ**: Backend / Runtime
- **ソース**: [Deno Blog](https://deno.com/blog)

## 概要

Deno チームは 2026-06-25 に **Deno 2.9** をリリースした。本リリースの目玉は、Web 技術（HTML/CSS/JS/TS）からネイティブデスクトップアプリを構築できる新サブコマンド **`deno desktop`** の追加である。加えて、npm/pnpm/yarn/Bun といった既存ツールチェーンからの「第一級マイグレーション」サポートが入り、他ランタイム・パッケージマネージャからの移行ハードルが大きく下がった。Node.js 26 互換も達成し、エコシステム互換性の面でも前進している。

5 月リリース（2.8 系）で `import defer`、`deno transpile` / `deno pack` / `deno bump-version` / `deno ci` / `deno why` / `deno audit fix` といった 6 つの新サブコマンド、Chrome DevTools でのネットワークデバッグ、npm cold install の約 3.66 倍高速化などが入っていた流れを継ぐ、機能拡張色の強いリリースとなっている。

## 主な変更点

- **`deno desktop`**: Web 技術ベースでネイティブデスクトップアプリを構築する仕組み。Electron/Tauri 的な領域に Deno が公式に踏み込む形。
- **第一級マイグレーション**: npm / pnpm / yarn / Bun のプロジェクトから Deno への移行を公式にサポート。
- **CSS モジュールインポート**: `import` 経由で CSS モジュールを扱えるように。
- **スナップショットテスト / パラメータ化テスト**: 組み込みテストランナーの表現力が向上。
- **`deno compile --bundle` のバイナリ縮小**: 単一実行ファイル配布時のサイズを削減。
- **Node.js 26 互換**: 最新の Node API に追従。

## 破壊的変更・移行ガイド

公式ブログ時点では大きな破壊的変更は強調されていない。マイナーリリース（2.x 系）のため、既存プロジェクトはそのままアップグレード可能と見られる。`deno desktop` や CSS モジュールインポートは新規追加機能であり、既存挙動への影響は限定的。Node.js 26 互換に伴う挙動差が気になる場合は、CI で `deno test` / `deno check` を回して回帰を確認するとよい。

## 今後の注目点

- `deno desktop` が Tauri / Electron の現実的な代替になり得るか。配布フロー・バイナリサイズ・OS ネイティブ API へのアクセス範囲が評価軸になる。
- npm/pnpm/yarn/Bun からのマイグレーション体験が、実プロジェクト規模でどこまでスムーズか。
- Node.js 26 互換の網羅度。ネイティブアドオンや一部 Node コア API の差異が残るかどうか。
- Bun との「バッテリー内蔵」路線の競争。Bun 1.3 の DB クライアント内蔵・ゼロコンフィグ路線に対し、Deno はデスクトップ・ツールチェーン整備で差別化を図る構図。

# Vue.js — v3.6 リリース

- **日付**: 2026-06-30
- **カテゴリ**: Frontend
- **ソース**: [VersionLog: Vue 3.6](https://versionlog.com/vuejs/3.6/), [Vue School: 2025 in Review](https://vueschool.io/articles/news/vue-js-2025-in-review-and-a-peek-into-2026/)

## 概要

Vue.js は 2026-06-11 に **v3.6** をリリースした。本リリースの中心テーマは **Vapor Mode の安定化**である。Vapor Mode は Virtual DOM のオーバーヘッドを排除し、コンポーネントを直接 DOM 操作へコンパイルする新しいコンパイル戦略で、ランタイム性能の向上とバンドルサイズの削減を狙う。これまで実験的位置づけだった機能が成熟フェーズに入り、本番投入の現実味が増した。

Vapor Mode は Solid.js 的な「コンパイル時に細粒度リアクティビティへ落とす」アプローチに近く、2026 年のフロントエンドフレームワーク全体が「fine-grained reactivity」「コンパイラ駆動の最適化」へ収斂していく潮流の一翼を担う。デモでは約 10 万コンポーネントを ~100ms でマウントするといった性能特性が示されてきた。

## 主な変更点

- **Vapor Mode の安定化**: Virtual DOM を介さず直接 DOM 操作へコンパイルする戦略が成熟。ランタイムのオーバーヘッドとバンドルサイズを削減。
- **段階的採用**: Vapor Mode はコンポーネント単位でオプトインする設計が想定されており、既存の Virtual DOM ベースのコンポーネントと混在可能。
- **コンパイラ駆動の最適化**: テンプレートをより効率的なコードへ変換し、再レンダリングコストを抑制。

## 破壊的変更・移行ガイド

3.x 系のマイナーリリースであり、既存アプリへの破壊的変更は基本的に意図されていない。Vapor Mode はオプトインのため、既存の Virtual DOM ベースのコードはそのまま動作する。Vapor Mode を採用する場合は、対象コンポーネントが Vapor の制約（直接 DOM コンパイルに伴う一部 API/パターンの非互換）に適合するかを個別に確認する必要がある。まずは性能がボトルネックになっているリスト描画・大量ノードのコンポーネントから段階的に試すのが現実的。

## 今後の注目点

- Vapor Mode の本番採用事例とベンチマーク。実アプリでどの程度のバンドル削減・描画高速化が得られるか。
- エコシステム（Nuxt、UI ライブラリ、devtools）の Vapor Mode 対応状況。
- React Compiler、Svelte 5 runes、Solid といった「コンパイラ駆動・細粒度リアクティビティ」勢との性能・DX 比較。
- Vapor Mode と既存 Virtual DOM コンポーネントの混在運用におけるベストプラクティスの確立。

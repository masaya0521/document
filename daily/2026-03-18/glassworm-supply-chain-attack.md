# GlassWorm — GitHub サプライチェーン攻撃の新たな脅威

- **日付**: 2026-03-18
- **カテゴリ**: Tech
- **ソース**: [The Hacker News](https://thehackernews.com/2026/03/glassworm-attack-uses-stolen-github.html)

## 概要

2026年3月、「GlassWorm」と呼ばれるマルウェアキャンペーンが GitHub 上の Python リポジトリを標的としたサプライチェーン攻撃を展開。盗まれた GitHub トークンを使って正規リポジトリにマルウェアを force-push する手法で、約150以上のリポジトリが侵害された。元々2025年10月に VS Code 拡張機能を標的として出現した GlassWorm が、より広範で検出困難な形で進化している。

## 詳細

### 攻撃の手法

1. **トークン窃取**: 2025年10月の VS Code（OpenVSX マーケットプレイス）攻撃で盗まれた GitHub クレデンシャルを利用
2. **リベース＆注入**: 対象リポジトリのデフォルトブランチで最新の正規コミットをリベースし、難読化された悪意のあるコードを追加
3. **Force-push**: 変更を force-push してリポジトリの履歴を上書き
4. **痕跡の最小化**: コミットメッセージと作者日付は元のコミットのまま維持し、コミッター日付のみが変更される

### 注入されるファイル

以下のような Python プロジェクトの一般的なエントリポイントが標的:
- `setup.py`
- `main.py`
- `app.py`

### 被害規模

- **侵害リポジトリ**: 3月3日〜9日の間に約150以上
- **対象プロジェクト**: Django アプリ、ML 研究コード、Streamlit ダッシュボード、PyPI パッケージ
- **VS Code 攻撃（2025年10月）**: 72の OpenVSX 拡張機能、推定35,000回以上のダウンロード
- **影響**: `pip install` やリポジトリのクローン＆実行でマルウェアが発動

### 検出が困難な理由

- コミット履歴が正規のものと見分けがつきにくい（作者日付が維持されるため）
- 難読化されたコードが既存ファイルに追記される形式
- 後続の「ForceMemo」攻撃では不可視 Unicode 文字を使ったさらに高度な隠蔽手法も確認

### 時系列

| 時期 | 出来事 |
|------|--------|
| 2025年10月 | GlassWorm が OpenVSX マーケットプレイスで72の VS Code 拡張機能を侵害 |
| 2025年10月〜2026年2月 | 盗まれた GitHub クレデンシャルが蓄積 |
| 2026年3月3日 | Python リポジトリへの force-push 注入が開始 |
| 2026年3月8日〜9日 | 大規模な注入活動のピーク |
| 2026年3月中旬 | セキュリティ研究者が検出・報告。「ForceMemo」と呼ばれる後続攻撃も確認 |

## 今後の注目点

- GitHub によるトークン無効化と影響を受けたリポジトリへの通知状況
- PyPI に公開された侵害パッケージの特定と削除の進捗
- VS Code / OpenVSX マーケットプレイスのセキュリティ強化策
- force-push を防ぐブランチ保護ルールの普及促進
- 開発者が取るべき対策: トークンのローテーション、ブランチ保護の有効化、依存関係の監査
- サプライチェーン攻撃の検出ツール（Sigstore、SLSA など）の採用加速

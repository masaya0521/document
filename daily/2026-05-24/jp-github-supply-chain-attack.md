# GitHub 内部リポジトリ不正アクセス事件：サプライチェーン攻撃の新たな脅威

- **日付**: 2026-05-24
- **カテゴリ**: Japan Tech
- **ソース**: [gihyo.jp](https://gihyo.jp/article/2026/05/github-internal-code-leak-potential)

## 概要

2026年5月20日、GitHub は内部リポジトリへの不正アクセスを調査中であることを公表した。攻撃グループ「TeamPCP」が、毒された VS Code 拡張機能を通じて GitHub 従業員の開発端末を侵害し、約3,800個の内部プライベートリポジトリにアクセスした。同時期に日本のフィンテック企業マネーフォワードも GitHub への不正アクセスを公表しており、開発インフラのセキュリティが改めて問われている。

## 詳細

### 攻撃の手法：毒された VS Code 拡張機能

攻撃の起点は、人気の VS Code 拡張機能「Nx Console」（nrwl.angular-console、バージョン18.95.0）のトロイの木馬化だった。Nx Console は220万インストールを誇る認証済みパブリッシャーの拡張機能で、このトロイの木馬版は2026年5月18日の UTC 12:30〜12:48 のわずか18分間だけ Visual Studio Marketplace で公開された。

この短い公開時間にもかかわらず、自動更新機能により複数の開発者の端末に配布された。拡張機能には以下の情報を窃取するクレデンシャルスティーラーが仕込まれていた：

- 1Password ボールト
- Anthropic Claude Code の設定情報
- npm、GitHub、AWS のアクセストークン
- IDE のローカル環境に保存された開発者シークレット

### GitHub 内部への侵入

窃取したクレデンシャルを使用し、攻撃者は GitHub の内部組織に属するプライベートリポジトリにアクセス。Copilot 関連、Enterprise Server、レッドチームモジュールなど約3,800個のリポジトリをクローンした。データは犯罪フォーラムで50,000ドル以上での販売が告知されている。

GitHub は、顧客リポジトリ・エンタープライズアカウント・ユーザーデータへの影響はなく、侵害は GitHub 自身の社内環境に限定されていると評価している。

### TeamPCP の正体

TeamPCP（UNC6780 として追跡）は、オープンソースのセキュリティツールや AI ミドルウェアを標的としたサプライチェーン攻撃を専門とするサイバー犯罪グループ。過去には Aqua の Trivy セキュリティスキャナー、CheckMarx の KICS、LiteLLM ライブラリ、Telnyx SDK、TanStack、MistralAI を侵害した実績がある。

### マネーフォワードへの影響

同時期の2026年5月1日、マネーフォワードも GitHub への不正アクセスを公表した。この事案では：

- 「マネーフォワード ビジネスカード」関連の370件の情報（カード保持者名・カード番号下4桁）が流出の可能性
- 本番データベースへの侵害や改ざんは確認されず
- 原因は個人情報を含むファイルが管理手順から外れて GitHub 上に保管されていたこと
- 銀行口座連携機能を一時停止する対応を実施（5月12日に順次再開）

### 「Megalodon」キャンペーンとの関連

同時期に GitHub プラットフォーム上で「Megalodon」と呼ばれる大規模サプライチェーン攻撃キャンペーンも発覚。6時間で5,561リポジトリに5,718件の悪意あるコミットを自動プッシュするという手法で、Laravel-Lang の PHP パッケージも認証情報窃取フレームワークの標的となった。

## 今後の注目点

- **IDE 拡張機能のサプライチェーンリスク**: VS Code 拡張機能が攻撃ベクターとして台頭。拡張機能のインストール・更新ポリシーの見直しが必要
- **開発者端末のゼロトラスト化**: 開発ツールに保存されたシークレットの管理強化が急務。1Password・AWS トークン等の IDE ローカル保存の危険性が顕在化
- **日本企業への波及**: マネーフォワード事案が示すように、GitHub をインフラとして利用する日本企業にとってもサプライチェーン攻撃は現実的な脅威
- **TeamPCP の次の標的**: AI 開発ツールチェーン（Claude Code 設定情報の窃取など）を標的に含めており、AI 開発を行う組織は特に警戒が必要
- **GitHub のセキュリティ対応**: 流出した内部コードに含まれる可能性のある脆弱性情報の悪用リスクと、GitHub の対策の推移

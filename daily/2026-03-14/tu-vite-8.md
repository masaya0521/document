# Vite 8.0 — Rolldown 統合による次世代ビルドツール

- **日付**: 2026-03（正式リリース）
- **カテゴリ**: Build Tool
- **ソース**: [Vite Blog](https://vite.dev/blog/announcing-vite8), [Vite 8 Beta](https://vite.dev/blog/announcing-vite8-beta)

## 概要

Vite 8.0 は Vite 2 以来最大のアーキテクチャ変更となるリリース。これまで開発時に esbuild、プロダクションビルドに Rollup という二重バンドラー構成だったものを、Rust ベースの統一バンドラー **Rolldown** に一本化した。これにより 10-30 倍のビルド高速化を実現しつつ、既存のプラグイン互換性を維持している。

## 主な変更点

### Rolldown 統一バンドラー

- 開発・プロダクション両方で Rolldown を使用
- Rollup 互換のプラグイン API をサポートし、既存プラグインの大半がそのまま動作
- 開発とプロダクションの挙動差異が大幅に削減

### パフォーマンス改善

- ビルド速度が 10-30 倍向上
- フルバンドルモードのサポート
- モジュールレベルの永続キャッシュ
- より柔軟なチャンク分割

### 新機能

- **統合 Devtools**: デバッグと分析のための組み込みツール
- **Wasm SSR サポート**: `.wasm?init` インポートが SSR 環境で動作
- **ブラウザコンソール転送**: ブラウザのコンソールログ・エラーを開発サーバーのターミナルに転送
- **Module Federation サポート**: マイクロフロントエンドアーキテクチャの標準的なサポート
- **tsconfig paths サポート**: `resolve.tsconfigPaths: true` で TypeScript のパスエイリアスを組み込みサポート（オプトイン）

### システム要件

- Node.js 20.19+ または 22.12+（Vite 7 と同じ）

## 破壊的変更・移行ガイド

Rolldown への移行に伴い、一部の挙動が変わる可能性がある：

1. **プラグイン互換性の確認**: 大半のプラグインはそのまま動作するが、Rollup の内部 API に依存するプラグインは更新が必要な場合がある
2. **バンドル出力の差異**: チャンク分割のアルゴリズムが変わるため、出力ファイルの構成が変わる可能性がある
3. **Node.js バージョン**: 20.19+ または 22.12+ が必要

## 今後の注目点

- **Vite+ (Alpha)**: VoidZero が開発中の次世代ツールチェーン
- **Rolldown の成熟**: 統一バンドラーとしてのさらなる最適化と機能追加
- **エコシステムの対応**: Tailwind CSS 4.2.1 など主要ツールの Vite 8 対応が進行中
- **永続キャッシュの改善**: モジュールレベルキャッシュの精度とパフォーマンスの向上

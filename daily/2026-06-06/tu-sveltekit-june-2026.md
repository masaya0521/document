# Svelte / SvelteKit — June 2026 マンスリーアップデート

- **日付**: 2026-06（月次まとめ）
- **カテゴリ**: Frontend
- **ソース**: [What's new in Svelte: June 2026](https://svelte.dev/blog/whats-new-in-svelte-june-2026)

## 概要

SvelteKit の「リモート関数（Remote Functions）」がこの数か月で大きく前進し、June 2026 のマンスリーアップデートでリモートクエリの非同期化が一区切りついた。`query.run()` の削除という破壊的変更を含む 2.61.0 が目玉で、イベントハンドラ・async コールバック・モジュールスコープのいずれからもリモートクエリを `await` できるようになった。あわせて Svelte 本体 5.56.0 ではマークアップ内での宣言が可能になり、language-tools が TypeScript 6.0 に対応した。

## 主な変更点

### SvelteKit のバージョン別ハイライト

- **2.57.0**: フォームの `submit` がブール値を返すよう改良
- **2.58.0**: `requested(...)` が `limit` パラメータを必須化し、戻り値の形式を変更
- **2.59.0**: `requested(...)` が `query.batch(...)` をサポート。`query.live(...)` を実験的に追加
- **2.60.0**: リモートフォームフィールドがブール値・数値を直接受け入れ可能に。未読の検証警告を追加
- **2.61.0**: `query.run()` を削除。クエリの非同期化と enhance コールバック仕様の変更

### リモートクエリの非同期化（2.61.0）

リモートクエリをイベントハンドラ、async コールバック、モジュールスコープから `await` できるようになった。キャッシュの重複排除（dedupe）がリアクティブ／非リアクティブ両方の利用箇所で共有されるため、同一クエリの多重発火を避けつつ任意の文脈から値を取得できる。

### query.live(...) の async-iterable 化

`query.live(...)` は長寿命のリモートクエリ購読を扱いやすくする API。2.59.0 で実験的に導入され、2.61.0 で async になり async-iterable として扱えるようになった。`for await...of` でストリーム的に購読を消費できる。

### Svelte 本体・エコシステム

- **Svelte 5.56.0**: マークアップ内で直接宣言が可能に
- **language-tools 0.18.0 / svelte2tsx 0.7.55**: TypeScript 6.0 対応
- **vite-plugin-svelte 7.1.0**: 開発環境でサーバーのオプティマイザーを有効化
- **mcp 0.1.23**: `stdio` モードにファイル読み込み機能を追加

## 破壊的変更・移行ガイド

- **`query.run()` の削除（2.61.0）**: 従来 `query.run()` で明示的に実行していた箇所は、リモートクエリの `await` ベースの新しい呼び出し方に置き換える必要がある。イベントハンドラ内などからは直接 `await query(...)` する形に移行する。
- **`requested(...)` の `limit` 必須化（2.58.0）**: `limit` パラメータ未指定だった呼び出しは引数を追加する必要がある。戻り値形式の変更にも追従が必要。

## 今後の注目点

- リモート関数（query / form / command）まわりの API は安定化に向けて継続的に仕様変更が入っているため、アップグレード時は CHANGELOG の破壊的変更を都度確認すること。
- `query.live(...)` の async-iterable 対応により、リアルタイム更新を伴う UI（通知・在庫・コラボ編集など）の実装パターンがどう定着するか。

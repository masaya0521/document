# Claude Mythos と日本のサイバーセキュリティ戦略

- **日付**: 2026-05-26
- **カテゴリ**: Japan Tech
- **ソース**: [ITmedia](https://www.itmedia.co.jp/aiplus/articles/2605/12/news102.html)

## 概要

Anthropic が2026年4月7日に発表した未公開フロンティアAIモデル「Claude Mythos Preview」は、ソフトウェア脆弱性の発見・悪用において人間の熟練セキュリティ研究者を凌駕する能力を持つとされ、一般公開は見送られた。この発表を受け、日本政府は5月12日にサイバー攻撃対策の強化を指示し、メガバンク3行がアクセス権取得に動くなど、日本のサイバーセキュリティ体制に大きな影響を与えている。

## Claude Mythos Preview の能力

Claude Mythos Preview は、既存のAIモデルとは次元の異なるセキュリティ能力を示した。

- **27年間放置されたOpenBSDの脆弱性**を自律的に発見
- **16年間のFFmpegバグ**（自動テスト500万回をすり抜けたもの）を検出
- **Linuxカーネルの複数脆弱性**を組み合わせた権限昇格攻撃を構築
- 英国AI安全研究所（UK AISI）が「全主要OS・ブラウザのゼロデイ脆弱性を自律的に発見・悪用する能力」を確認

これらの能力が攻撃者に渡るリスクを踏まえ、Anthropic は一般公開を停止し、防御目的の限定提供に切り替えた。

## Project Glasswing

Anthropic は防御利用に限定した企業連合「Project Glasswing」を組織し、Mythos Preview を用いたOSSの脆弱性スキャンを実施している。

- **参加企業**: AWS、Apple、Google、Microsoft、NVIDIA、CrowdStrike、Palo Alto Networks など約50組織
- **成果**: 1,000以上のOSSプロジェクトから6,202件の高・重大脆弱性を検出。Cloudflare で2,000件、Firefox で271件のバグを特定
- **支援規模**: 最大1億ドルのMythos Preview利用クレジット、400万ドルのOSSセキュリティ組織への寄付

## 日本政府の対応

### 高市首相のサイバー攻撃対策指示（5月12日）

高市早苗首相は閣僚懇談会で、Claude Mythosなど最新AIのサイバー攻撃能力向上を受け、松本尚デジタル大臣を中心とした対策を指示した。

- 重要インフラ事業者との連携強化
- セキュリティ脆弱性の「発見と修正」に重点を置いた対策
- 政府として対策パッケージの早急な策定

内閣官房の国家サイバー統括室（NCO）が司令塔となるが、**日本政府は現時点でMythosへのアクセス権を持っていない**。松本大臣は「Mythosが無いという前提で対策を進めなくてはならない」と述べ、米国側との交渉が進行中であることを明かした。

### メガバンク3行のアクセス権取得

5月13日の報道によると、三菱UFJ銀行・三井住友銀行・みずほ銀行の3メガバンクが約2週間以内にClaude Mythos Previewへのアクセス権を取得する見通し。日本企業として初の正式活用事例となる見込みで、金融インフラの防御にAIを活用する先行事例として注目される。

## 構造的な課題

この事態は、日本のサイバーセキュリティにおける構造的な課題を浮き彫りにしている。

1. **AIセキュリティの非対称性**: 攻撃側がAIを活用するスピードに対し、防御側の対応が遅れるリスク
2. **米国依存**: 最先端のセキュリティAIへのアクセスが米国企業・政府の意思決定に左右される
3. **民間先行**: 政府よりもメガバンクの方がアクセス権取得で先行するという逆転現象
4. **国産AI基盤との連携**: 政府AI「源内」との棲み分けと連携をどう設計するか

# 富士通AI駆動ソフトウェア開発プラットフォーム — 100倍の生産性向上の実態

- **日付**: 2026-02-20
- **カテゴリ**: Japan Tech
- **ソース**: [Fujitsu Global](https://global.fujitsu/en-global/pr/news/2026/02/17-01) / [Japan Industry News](https://www.japanindustrynews.com/2026/02/fujitsu-launches-ai-driven-software-development-platform/)

## 概要

富士通は2026年2月17日、ソフトウェア開発ライフサイクル全体を自動化する「AI駆動ソフトウェア開発プラットフォーム」を発表した。同社がCohere社と共同開発した大規模言語モデル「Takane」とエージェントAI技術を組み合わせ、要件定義から設計・実装・統合テストまでの全工程を人手を介さずに実行する。概念実証（PoC）では、従来3人月を要した開発作業を4時間で完了し、100倍の生産性向上を実証した。

## 技術的な特徴

### Takane LLM

- 富士通とCohere社が共同開発した大規模言語モデル
- 日本語の法令・規制文書の理解に特化した設計
- 医療・金融・行政などの複雑な業務システムの文脈を理解する能力

### エージェントAI技術

- 複数のAIエージェントが協調して各開発フェーズを逐次実行
- 「AI-Ready Engineering」と呼ぶアプローチで、既存システムの資産（ドキュメント、コード、仕様書等）をAIが正確に理解できる形に整備
- エージェントが既存システムの構造を把握した上で、変更要求に対する最適な実装を自動生成

### 自動化の範囲

1. **要件定義**: 法令改正や業務要件の変更内容を解析
2. **設計**: 既存システムへの影響分析と設計書の自動生成
3. **実装**: ソースコードの自動生成・修正
4. **統合テスト**: テストケースの自動生成と実行

## PoCの詳細

### 対象: 診療報酬改定対応

- 日本の医療制度では2年ごとに診療報酬が改定され、医療関連ソフトウェアの大規模な修正が必要となる
- 従来、1件の改定対応に3人月（約480時間）の開発工数が必要
- AI駆動プラットフォームを適用したところ、同じ対応を4時間で完了
- **100倍の生産性向上**を実証

### 2026年1月より実運用開始

医療関連ソフトウェアの診療報酬改定対応から実運用を開始。

## 業界への影響

### 日本のSI業界への示唆

- 日本のSI（システムインテグレーション）業界は、法令改正対応や保守運用に多大な人的リソースを投入してきた
- AIによる自動化が本格化すれば、SIerのビジネスモデルの根本的な変革が求められる
- 一方で、「AI-Ready Engineering」の整備作業自体が新たなコンサルティング需要を生む可能性

### グローバルな文脈

- Microsoft GitHub Copilot、Google Gemini Code Assistなどコード生成AIは既に普及しているが、富士通のアプローチは「開発ライフサイクル全体の自動化」という点で差別化
- 特に日本語の法令・規制文書を扱える点は、日本市場における大きなアドバンテージ

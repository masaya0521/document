# Go 1.26 — Green Tea GC と主要な新機能

- **日付**: 2026-02-10
- **カテゴリ**: Language
- **ソース**: [公式リリースブログ](https://go.dev/blog/go1.26), [リリースノート](https://go.dev/doc/go1.26), [Green Tea GC ブログ](https://go.dev/blog/greenteagc)

## 概要

Go 1.26 は 2026-02-10 にリリースされた Go の最新メジャーリリース。最大の目玉は **Green Tea ガベージコレクタ**のデフォルト有効化で、実世界のワークロードで GC オーバーヘッドを 10〜40% 削減する。加えて、言語仕様の小さな拡張、`go fix` コマンドの全面刷新、標準ライブラリへの新パッケージ追加など、開発者体験を大きく向上させるアップデートが含まれる。

## 主な変更点

### Green Tea ガベージコレクタ

Go 1.25 で実験的に導入された Green Tea GC が、フィードバックを経てデフォルト有効化された。

- **GC オーバーヘッド 10〜40% 削減**: GC を多用するワークロードで顕著な効果。多くのケースで約 10% 削減、一部ワークロードでは最大 40% の削減
- **設計思想**: 個々のオブジェクトを個別にスキャンする代わりに、8 KiB の連続メモリブロック（span）単位でスキャンする。特に 512 バイト以下の小さなオブジェクト（最も一般的かつスキャン効率が低い）に効果的
- **ベクトル化命令**: amd64 アーキテクチャでは、十分なオブジェクトが蓄積された場合にベクトル化 CPU 命令を使用してメモリスパンを一括処理
- **背景**: GC コストの約 90% はマーキングに費やされ、そのうち少なくとも 35% はヒープメモリアクセスのストールで占められている。Green Tea はこのボトルネックをメモリの局所性向上と CPU スケーラビリティの改善で解消

### 言語仕様の変更

1. **`new` 関数の拡張**: `new` 関数が任意の値を引数に取れるようになった。これにより初期値を持つ変数の作成が 1 ステートメントで可能に
2. **ジェネリック型の自己参照**: ジェネリック型が自身の型パラメータリスト内で自己参照できるようになり、複雑なデータ構造やインターフェースの表現力が向上

### `go fix` の全面刷新

`go fix` サブコマンドが完全に書き直された。

- **24 個のモダナイザー**: コードベースを自動的にモダンな Go のイディオムに更新するアルゴリズム群を搭載
- **gopls 統合**: モダナイザーは gopls にも統合されており、エディタ上でリアルタイムにフィードバックを受けられる
- **ソースレベルインライナー**: `//go:fix inline` ディレクティブを持つ関数を考慮する新しいインラインアナライザを搭載
- **一括適用**: `go fix` コマンドで複数パッケージを一度にモダナイズ可能

### 標準ライブラリの新パッケージ

- `crypto/hpke` — Hybrid Public Key Encryption
- `crypto/mlkem/mlkemtest` — ML-KEM テストユーティリティ
- `testing/cryptotest` — 暗号テスト支援

### セキュリティ修正（Go 1.25.7 / 1.24.13）

2026-02-04 には Go 1.25.7 と 1.24.13 もリリースされ、`go` コマンドと `crypto/tls` パッケージのセキュリティ修正、`crypto/x509` のバグ修正が含まれる。

## 今後の注目点

- **Green Tea GC のベクトル拡張**: ベンチマークでは追加で約 10% の GC CPU 使用量削減が見込まれており、今後のリリースでさらなる最適化が期待される
- **Go 1.27（2026 年 8 月頃予想）**: 次期リリースでの言語・ランタイム改善の方向性
- **`go fix` エコシステム**: モダナイザーの追加により、レガシーコードの自動移行がさらに容易になる見込み

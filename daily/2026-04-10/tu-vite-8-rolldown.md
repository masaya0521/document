# Vite 8.0 — Rolldown 統合による統一バンドラーアーキテクチャ

- **日付**: 2026-03-12
- **カテゴリ**: Build Tool
- **ソース**: [Vite Blog](https://vite.dev/blog/announcing-vite8), [The Register](https://www.theregister.com/2026/03/16/vite_8_rolldown/)

## 概要

Vite 8.0 は、開発用の esbuild と本番用の Rollup という二つのバンドラーを、Rust 製の統一バンドラー Rolldown に置き換えた。これにより開発環境と本番環境の動作差異が解消され、ビルド速度が 10-30x 向上した。Linear では 46秒→6秒、Beehiiv では 64% の短縮が報告されている。

## 主な変更点

### Rolldown 統合

- **統一バンドラー**: esbuild（開発用トランスフォーム）と Rollup（本番用バンドル）を Rolldown に統一
- **プラグイン互換性**: 既存の Rollup/Vite プラグインとの完全な互換性を維持
- **dev/prod の一貫性**: 開発環境と本番環境で同一のバンドラーを使用し、動作差異を解消

### 新機能

- **Vite Devtools**: デバッグと分析のための統合開発ツール
- **`resolve.tsconfigPaths`**: TypeScript のパス設定を自動解決（外部プラグイン不要）
- **デコレータメタデータサポート**: 外部プラグインなしでデコレータメタデータを利用可能
- **WebAssembly SSR 対応**: サーバーサイドレンダリング環境での WASM 動作をサポート
- **コンソールログ転送**: ブラウザのログを開発サーバーターミナルに出力

### パフォーマンス

| メトリクス | 改善 |
|-----------|------|
| ビルド速度 | 10-30x 高速化 |
| Linear 事例 | 46秒 → 6秒 |
| Beehiiv 事例 | 64% 短縮 |

### インストールサイズ

Vite 8 は Vite 7 と比較して約 15MB 増加。内訳は lightningcss（約10MB）と Rolldown（約5MB）。

## 破壊的変更・移行ガイド

### Node.js 要件

- **必須バージョン**: Node.js 20.19+ または 22.12+
- Node.js 18 はサポート外

### 移行手順

1. **Node.js バージョンを確認**: 20.19+ または 22.12+ に更新
2. **互換性レイヤーの活用**: ほとんどのプロジェクトは設定変更なしでアップグレード可能
3. **段階的移行（推奨）**: 複雑なプロジェクトでは Vite 7 環境で `rolldown-vite` パッケージを先に試す
4. **プラグイン互換性の確認**: Rollup プラグインは概ね互換性があるが、内部 API に依存するプラグインは検証が必要

## 今後の注目点

- **Rolldown 1.0 正式リリース**: 2026年1月に RC が公開済み。Vite 8 での実運用フィードバックを反映して正式版へ
- **Oxc との統合深化**: Rolldown と同じ Rust ツールチェーンの Oxc（パーサー・リンター）との統合がさらに進む見込み
- **Vite エコシステムの拡大**: Vitest、Nuxt、SvelteKit など Vite ベースのツールが順次 Rolldown 対応
- **esbuild の位置づけ**: Vite のコアからは外れたが、単体ツールとしては引き続きメンテナンスされる

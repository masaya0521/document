# SMBC・富士通・ソフトバンク「国産ヘルスケア基盤」: 6,000万人規模の医療DX構想

- **日付**: 2026-05-25
- **カテゴリ**: Japan Tech
- **ソース**: [ソフトバンク プレスリリース](https://www.softbank.jp/corp/news/press/sbkk/2026/20260519_01/)

## 概要

SMBCグループ、富士通、ソフトバンクの3社は2026年5月18日、健康・医療分野での業務提携に関する基本合意書を締結した。国民皆保険制度を基盤とする日本の医療を持続可能なものとするため、「国産ヘルスケア基盤」を構築し、6,000万人規模の利用者と4,000医療機関への導入を目指す。事業開始は2026年10月を予定しており、将来的な医療費増加幅の5兆円規模の抑制を目標に掲げる。

## 詳細

### 提携の背景: 日本の医療が直面する課題

日本の医療費は年間45兆円を超え、高齢化の進行に伴い増加の一途をたどっている。一方で、医療従事者の不足、地域間の医療格差、電子カルテの分断によるデータサイロ化など、構造的な課題が山積している。特に電子カルテについては、富士通自身が「データ分断を招いた」と自省を表明しており、この反省が今回の提携の出発点の一つとなっている。

### 3社の役割分担

**SMBCグループ（金融・普及基盤）**
- 提携に基づくサービスの幅広い普及を担当
- ヘルスケアと金融の連携・融合によるサービス価値向上
- 既に2026年3月から「Oliveヘルスケア」を提供開始しており、これを発展させる

**富士通（技術基盤・医療AI）**
- データプラットフォームの構築・管理
- 医療機関向けAIの構築
- 医療データを用いた創薬研究向けの次世代計算資源・基盤の開発
- 電子カルテ大手としての知見と反省を活かしたデータ統合

**ソフトバンク（ユーザー接点・国産AI）**
- PayPay・LINE・Yahoo! JAPAN などのグループ経済圏を活用
- ソブリンクラウドおよび国産LLMを活用した国内完結型ユーザーアプリの開発
- 医療データの国内処理を保証する技術基盤の提供

### 技術的アーキテクチャ

本基盤の特徴は、ソブリンクラウドと国産LLMの採用による「データ主権」の確保にある。医療データという極めて機微な個人情報を扱うにあたり、海外クラウドへのデータ流出リスクを排除する設計としている。

個々人に寄り添う「健康パートナーAIエージェント」を構築し、ユーザーアプリを通じて提供する計画で、予防医療から治療後のフォローアップまでを一貫して支援する構想となっている。

### 目標数値

| 指標 | 目標 |
|------|------|
| 利用者規模 | 6,000万人 |
| 導入医療機関数 | 4,000 |
| 医療費抑制効果 | 5兆円規模 |
| 事業開始 | 2026年10月 |

## 今後の注目点

- 2026年10月の事業開始時にどの程度の機能が実装されるか
- 電子カルテの分断問題を実際にどう解決するか（既存ベンダーとの調整が鍵）
- 6,000万人の利用者獲得に向けた具体的な普及戦略（PayPay・LINE経由のオンボーディング）
- 医療データのプライバシー保護と利活用のバランス（国産LLMの性能面での課題）
- 競合する医療DXプラットフォームとの差別化と医療機関側の受容性
- 政府の医療DX政策（マイナ保険証など）との連携・整合性

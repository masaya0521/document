# Megalodon サプライチェーン攻撃: GitHub CI/CD を標的にした大規模攻撃の全貌

- **日付**: 2026-05-25
- **カテゴリ**: Tech
- **ソース**: [The Hacker News](https://thehackernews.com/2026/05/megalodon-github-attack-targets-5561.html)

## 概要

2026年5月18日、「Megalodon」と名付けられた大規模サプライチェーン攻撃が発生した。わずか6時間の間に 5,718件の悪意のあるコミットが 5,561の GitHub リポジトリに push され、CI/CD パイプラインを通じてクラウド認証情報やシークレットが大量に窃取された。攻撃グループ「TeamPCP」による組織的な犯行とされ、GitHub Actions のワークフローを悪用した史上最大級のサプライチェーン攻撃となった。

## 詳細

### 攻撃タイムライン

- **2026年5月18日 11:36-17:48 UTC**: 約6時間で 5,718コミットを 5,561リポジトリに push
- 攻撃者は使い捨ての GitHub アカウント（ランダムな8文字のユーザー名: rkb8el9r, bhlru9nr 等）を使用
- git config を偽装し、4つの偽名（build-bot, auto-ci, ci-bot, pipeline-bot）と7種類のコミットメッセージを使い回してCI メンテナンスを装った

### 攻撃手法

2つのペイロード変種が確認されている:

1. **SysDiag（大量散布型）**: すべての push と pull request をトリガーとする新規ワークフローを追加。マージされた瞬間に自動実行される
2. **Optimize-Build（標的型）**: 既存ワークフローを `workflow_dispatch` トリガーに置換。攻撃者が GitHub API 経由で任意のタイミングで起動できる dormant backdoor を作成

ペイロードは Base64 エンコードされた bash スクリプトで、以下を C2 サーバー（216.126.225.129:8443）に送信:
- AWS / Google Cloud 認証情報
- SSH 鍵
- OIDC トークン
- GitHub Actions シークレット
- Docker / Kubernetes 設定

### 侵入経路

Hudson Rock の調査により、影響を受けたリポジトリのユーザーの **31%以上が infostealer マルウェアに感染**していたことが判明。盗まれた GitHub 認証情報（PAT やデプロイキー）が大規模攻撃を可能にした。

### 被害規模

- 5月21日時点で C2 サーバーには **575,352ファイル、449GB** のデータが蓄積
- 3分間の観測だけで 2,365ファイル、12GB が追加される速度
- npm パッケージ `@tiledesk/tiledesk-server`（バージョン 2.18.6-2.18.12）にも標的型変種が混入。正規メンテナが汚染されたソースからそのまま npm publish したことで拡散

### 対応状況

- npm は バイパス 2FA トークンを無効化し、Trusted Publishing への移行を推奨
- GitHub は影響を受けたリポジトリの特定と通知を進行中
- セキュリティ各社（SafeDep, StepSecurity, OX Security 等）が検出ルールと対策ガイダンスを公開

## 今後の注目点

- **infostealer 対策の重要性**: 開発者の認証情報窃取がサプライチェーン攻撃の起点となるパターンが定着。ハードウェアセキュリティキーの導入が急務
- **CI/CD ワークフローの監査**: 外部からの PR や見慣れないワークフロー変更を自動検出する仕組みの必要性
- **npm / PyPI 等パッケージレジストリの防御**: 正規メンテナのリポジトリが汚染されるケースへの対応が課題
- **GitHub の対応策**: GitHub がプラットフォームレベルでどのような再発防止策を講じるかに注目
- **TeamPCP の動向**: 窃取された 449GB のデータがどう悪用されるか、今後のサプライチェーン攻撃への波及

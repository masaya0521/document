# Node.js — 年1回メジャーリリースへの移行

- **日付**: 2026-06-08
- **カテゴリ**: Backend
- **ソース**: [Node.js Moves to One Major Release Per Year — InfoQ](https://www.infoq.com/news/2026/06/nodejs-release-changes/), [Evolving the Node.js Release Schedule — Node.js Blog](https://nodejs.org/en/blog/announcements/evolving-the-nodejs-release-schedule)

## 概要

Node.js プロジェクトが、長年続けてきた「年2回メジャーリリース・奇数版と偶数版」のモデルを廃止し、**年1回のメジャーリリース**へ移行することを発表した。Node.js 26 が現行モデルにおける最後のリリースとなり、Node.js 27 から新方式が適用される。すべてのリリースが LTS となり、これまで企業導入が敬遠しがちだった奇数版（非 LTS）の扱いが消える。メンテナの負荷軽減と、現代のエコシステムの実態に合わせたサイクルの最適化が狙い。

## 主な変更点

- **リリース頻度**: 年2回 → **年1回**のメジャーリリース。
- **LTS 化**: すべてのリリースが LTS になり、奇数版・偶数版の区別を廃止。
- **Alpha チャネルの導入**: 新機能テスト用に、メジャーリリース前 **6ヶ月間の Alpha プレリリース版**を提供。
- **バージョン番号と年の一致**: メジャーバージョン番号が初回リリース年に対応する。
  - Node.js 27 → 2027年系
  - Node.js 28 → 2028年系
- **スケジュール**: Node.js 27 が **2026年10月**に最初のリリースとして登場し、**2027年4月**以降は年1回（4月にメジャー、その後 LTS 昇格）の運用へ。

## 変更理由

主たる動機はメンテナ負荷の軽減。複数リリースラインの並行管理、修正のバックポート、そして採用率が低い奇数版（非 LTS）への対応が継続的な負担になっていた。提唱者の James Snell は「現在の計画は約10年前の企業導入サイクルを前提に設計されたもので、現在のエコシステムのニーズとは乖離している」と指摘している。

## 破壊的変更・移行ガイド

- 破壊的な API 変更そのものではなく、**バージョン採用戦略の見直し**が必要になる。
- 奇数版を「お試し版」として使い分けていたチームは、今後は Alpha チャネルで新機能を先行検証する運用に切り替える。
- CI のマトリクスや Docker ベースイメージのタグ運用（`node:26` 等）は、年1回サイクル前提に整理し直すとよい。
- 直近では Node.js 26.3.0（2026年6月1日）が Current ライン。同日に Node.js 25 が EOL を迎えている点にも注意。

## 今後の注目点

- 2026年10月の Node.js 27 リリースが、新スケジュールの実質的なスタート。
- 「全リリース LTS」により、バージョン選定が単純化される一方、各メジャーのサポート期間の具体的な定義（Active/Maintenance の境界）に注目。
- Alpha チャネルがエコシステム（ライブラリ作者・ツールチェーン）にどの程度活用されるかが、新方式の成否を左右する。

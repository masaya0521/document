# Go — 1.26（Green Tea GC のデフォルト化）

- **日付**: 2026-06-05（1.26.0 リリース: 2026-02 / 最新パッチ 1.26.4: 2026-06-02）
- **カテゴリ**: Language
- **ソース**: [Go 1.26 Release Notes](https://go.dev/doc/go1.26), [Go 1.26 is released](https://go.dev/blog/go1.26), [The Green Tea Garbage Collector](https://go.dev/blog/greenteagc)

## 概要

Go 1.26 は 2026年2月にリリースされたメジャーバージョンで、最大の目玉は **Green Tea ガベージコレクタのデフォルト有効化**。Go 1.25 で実験的に提供されていたものがフィードバックを取り込んで標準となり、GC を多用する実プログラムで **GC オーバーヘッドを 10〜40% 削減**する。

2026年6月2日にはセキュリティ修正を含むパッチ 1.26.4（および旧系列の 1.25.11）がリリースされており、本番環境では最新パッチへの追従が推奨される。

## 主な変更点

### Green Tea GC

- 小さなオブジェクトのマーキング・スキャンを、ローカリティと CPU スケーラビリティの改善により高速化
- 実プログラムで GC オーバーヘッドを 10〜40% 削減。新しい AMD64 系 CPU ではさらに約 10% の改善が見込まれる
- 既存の並行コレクタの「書き直し」ではなく、ペーシング・CPU オーバーヘッド・ヒープサイジングの洗練
- 一時的なオプトアウトとしてビルド時に `GOEXPERIMENT=nogreenteagc` を指定可能（ただし Go 1.27 でオプトアウトは廃止予定）

### ランタイム・性能

- cgo 呼び出しのベースラインオーバーヘッドを約 30% 削減
- 64-bit プラットフォームで起動時にヒープのベースアドレスをランダム化（cgo 利用時のセキュリティ強化）
- コンパイラがより多くの状況でスライスのバッキングストアをスタック上に確保し、性能向上

### 言語・標準ライブラリ

- 組み込み `new` 関数がオペランドに式を取れるようになり、変数の初期値を指定可能に
- ジェネリック型が自身の型パラメータリスト内で自己参照できるように
- `crypto/hpke` パッケージを新設（RFC 9180 の HPKE 実装、ポスト量子ハイブリッド KEM 対応）
- 実験的な `simd/archsimd` パッケージを追加（amd64 で 128/256/512-bit ベクタ型をサポート）

## 破壊的変更・移行ガイド

- Go 1.26 は **macOS 12 Monterey で動作する最後のリリース**。Go 1.27 は macOS 13 Ventura 以降が必要
- 壊れていた 32-bit windows/arm ポートを削除
- Green Tea GC は基本的に透過的だが、GC 挙動に敏感なワークロードはベンチマークで確認し、問題があれば一時的に `GOEXPERIMENT=nogreenteagc` で切り戻し可能

## 今後の注目点

- Go 1.27（2026年8月予定）で Green Tea GC のオプトアウトが廃止される見込み
- `simd/archsimd` の正式化動向と、数値計算・暗号処理での活用
- ポスト量子暗号（HPKE ハイブリッド KEM）対応の広がり

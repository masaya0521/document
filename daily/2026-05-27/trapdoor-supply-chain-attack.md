# TrapDoor: npm・PyPI・Crates.io を横断するサプライチェーン攻撃

- **日付**: 2026-05-27
- **カテゴリ**: Tech
- **ソース**: [The Hacker News](https://thehackernews.com/2026/05/trapdoor-supply-chain-attack-spreads.html)

## 概要

2026年5月22日、npm・PyPI・Crates.io の3つのパッケージエコシステムを横断する大規模サプライチェーン攻撃キャンペーン「TrapDoor」が検出された。34以上の悪意あるパッケージが384以上のバージョンにわたって公開され、暗号通貨・DeFi・AI 関連の開発者コミュニティを標的にしている。AI コーディングアシスタントの設定ファイルを悪用するという新しい攻撃手法も確認され、ソフトウェアサプライチェーンセキュリティの新たな脅威として注目されている。

## 詳細

### 攻撃手法

TrapDoor は各パッケージエコシステムの特性を巧みに利用している。

**Python（PyPI）パッケージ:**
- `import` 時に自動実行されるコードを仕込む
- 攻撃者が管理する GitHub Pages ドメインから JavaScript をダウンロードし、`node -e` で実行
- Python パッケージからリモートの JavaScript ペイロードに実行を委任することで、公開後も攻撃内容を柔軟に変更可能

**Rust（Crates.io）パッケージ:**
- Rust のコンパイル時に自動実行される `build.rs` を悪用
- ローカルのキーストアを検出し、ハードコードされた鍵 `cargo-build-helper-2026` で XOR 暗号化
- 暗号化したデータを GitHub Gists に送出

**npm パッケージ:**
- `prompt-engineering-toolkit`、`solidity-deploy-guard`、`defi-threat-scanner` など、開発者ツールに偽装した名前を使用
- インストール・ビルド時のスクリプトフックを利用して悪意あるコードを実行

### 窃取対象データ

- SSH 鍵
- 暗号通貨ウォレットのキーストア（Sui / Solana / Aptos）
- AWS 認証情報
- GitHub トークン
- ブラウザのログインデータベース
- 暗号通貨ウォレットのブラウザ拡張データ
- 環境変数・API キー
- ローカル開発設定ファイル

### AI アシスタントを悪用する新手法

TrapDoor の特に注目すべき点は、AI コーディングアシスタントの設定ファイルを攻撃ベクトルとして利用していることだ。

攻撃者は `.cursorrules` や `CLAUDE.md` に隠し命令を埋め込み、AI アシスタントに「セキュリティスキャン」を実行させることで、シークレットの発見と送出を行う。具体的には、`browser-use/browser-use`、`langchain-ai/langchain`、`langflow-ai/langflow` などの人気 AI・開発者プロジェクトに対して GitHub PR を作成し、これらの悪意ある設定ファイルの混入を試みている。

### 検出と対応

セキュリティ企業 Socket が TrapDoor のリリースを**中央値5分27秒**で検出し、広範な被害の拡大前にキャンペーン全体を悪意あるものとして分類することに成功した。

## 今後の注目点

- **AI アシスタントの設定ファイルのセキュリティ**: `CLAUDE.md` や `.cursorrules` などの AI 設定ファイルが新たな攻撃対象となった。プロジェクトへの PR を通じてこれらのファイルが改竄されるリスクへの対策が必要
- **クロスエコシステム攻撃の増加**: npm・PyPI・Crates.io を同時に標的にする手法は、単一エコシステムの対策では防げない。組織横断的なサプライチェーン監視の重要性が高まる
- **ビルドシステムの暗黙の信頼**: Rust の `build.rs` や Python の `import` 時自動実行など、ビルドプロセスにおける暗黙的なコード実行を再考する動きが加速する可能性
- **パッケージレジストリの対策強化**: Socket のような高速検出システムの需要増加と、各レジストリ側での公開前スキャン強化が期待される

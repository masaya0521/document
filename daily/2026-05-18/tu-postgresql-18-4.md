# PostgreSQL — 18.4 / 17.10 / 16.14 / 15.18 / 14.23

- **日付**: 2026-05-18
- **カテゴリ**: DB・Infra
- **ソース**: [PostgreSQL News](https://www.postgresql.org/about/news/postgresql-184-1710-1614-1518-and-1423-released-3297/), [18.4 Release Notes](https://www.postgresql.org/docs/18/release-18-4.html)

## 概要

PostgreSQL Global Development Group は 2026 年 5 月 14 日、現在サポートされている全ブランチに対するアップデートをリリースした。11 件のセキュリティ脆弱性と 60 件超のバグ修正を含み、特にスーパーユーザー権限での任意ファイル上書きや任意コード実行に至る可能性のある重大な CVE を複数含む。18.X 系では dump/restore は不要で、バイナリ更新のみで適用できる。

## 主な変更点

### セキュリティ修正（11件）

主要な CVE は以下：

- **CVE-2026-6472** — `CREATE TYPE` の権限確認漏れにより、攻撃者が他のクエリを乗っ取れる可能性。
- **CVE-2026-6473** (CVSS 8.8) — メモリ確保時の整数オーバーフローでメモリ外書き込みが発生し、segfault に到達する。
- **CVE-2026-6475** (CVSS 8.8) — `pg_basebackup` と `pg_rewind` がシンボリックリンクをたどり、スーパーユーザーによる任意ファイルの上書きを許す。
- **CVE-2026-6477** (CVSS 8.8) — `libpq` の `lo_*` 関数で、悪意あるサーバがクライアントスタックのメモリを上書きできる。
- **CVE-2026-6637** (CVSS 8.8) — contrib モジュール `refint` のスタックバッファオーバーフローによる任意コード実行。

このほか、悪意あるクライアントが SSL と GSS の暗号化リクエストを交互に拒否させ続けることで接続済み backend をクラッシュさせる DoS など、計 11 件を修正した。

### 主要なバグ修正

- 非決定的照合順序での一意インデックス利用時に **クエリが誤った結果を返す問題** を修正。
- **外部キーが意図せず非遅延化（DEFERRABLE が失われる）** 不具合を修正。
- **パーティションプルーニング** におけるプランナーの適用判定を改善。
- **仮想生成列（virtual generated column）** に関する複数の問題を解消。
- 論理レプリケーションのスロット同期ワーカーがハングする問題を修正。
- タイムゾーンデータを tzdata 2026b に更新（ブリティッシュコロンビア州と Moldova の DST 修正）。

## 破壊的変更・移行ガイド

破壊的変更はなし。18.X からのアップグレードは `pg_upgrade` や dump/restore を行わずに済むため、運用上の負担は小さい。ただし「PostgreSQL のアップデートは累積的」のため、すべてのバグ・セキュリティ修正を反映するには **最新マイナーへの更新が必須** 。手順は以下の通り：

1. PostgreSQL を停止
2. パッケージマネージャ等でバイナリを更新
3. PostgreSQL を再起動

`refint` の CVE-2026-6637 は contrib モジュール側のため、影響を受けるのは `refint` 利用環境に限られるが、念のためインストール状況を確認するとよい。`pg_basebackup` / `pg_rewind` の CVE-2026-6475 はバックアップ運用に直結するため優先度が高い。

## 今後の注目点

- PostgreSQL 18 系のサポートは引き続き続く（メジャー新機能は通常秋にリリースされる）。
- 直近では 13.X 系の EOL（2025 年 11 月）が完了済みであり、14.X 系が最古サポート対象。本リリースの 14.23 を最後に近づくバージョンで運用しているチームは、メジャーアップグレードの計画を始める時期。
- CVE-2026-6475 と CVE-2026-6477 は外部接続を伴う系統的な脆弱性のため、マネージド DB プロバイダ（RDS, Cloud SQL, Supabase 等）のパッチ反映状況も確認しておきたい。

# GlassWorm サプライチェーン攻撃 — 開発者を標的にした新世代の脅威

- **日付**: 2026-03-17
- **カテゴリ**: Tech
- **ソース**: [The Hacker News](https://thehackernews.com/2026/03/glassworm-attack-uses-stolen-github.html), [Aikido](https://www.aikido.dev/blog/glassworm-returns-unicode-attack-github-npm-vscode)

## 概要

2026年3月初旬から、「GlassWorm」と呼ばれる大規模なサプライチェーン攻撃が開発者コミュニティを襲っている。盗まれた GitHub トークンを使って150以上のリポジトリにマルウェアを注入し、不可視 Unicode 文字と Solana ブロックチェーンを悪用する新手法が特徴。Python リポジトリ、npm パッケージ、VS Code / Cursor 拡張機能と、開発者のツールチェーン全体が標的となっている。

## 詳細

### 攻撃の4段階

**第1段階: 初期感染**
- 悪意のある VS Code / Cursor エクステンションを通じて GlassWorm マルウェアでシステムを侵害
- GitHub トークンやその他の認証情報を盗取

**第2段階: リポジトリ侵害（ForceMemo 攻撃）**
- 盗まれた認証情報を利用して、ターゲットリポジトリのデフォルトブランチに悪意のある変更を強制プッシュ
- 元のコミットメッセージ、作成者、作成日時をそのまま保持するため、追跡が極めて困難

**第3段階: コード埋め込み**
- `setup.py`、`main.py`、`app.py` などのファイルに難読化されたマルウェアコードを追加
- 不可視 Unicode 文字（ゼロ幅文字等）を使い、コードレビューでの発見を回避

**第4段階: ペイロード実行**
- Base64 エンコードされたペイロードが Solana ウォレットのトランザクションメモフィールドを照会してペイロード URL を抽出
- ブロックチェーンをC2（Command & Control）インフラとして悪用することで、従来のドメインベースのテイクダウンを回避

### 影響範囲

- **GitHub リポジトリ**: 151以上（削除済みを含めるとさらに多い）
- **対象プロジェクト**: Django アプリ、機械学習研究コード、Streamlit ダッシュボード、PyPI パッケージ
- **npm パッケージ**: 複数が汚染
- **VS Code / Open VSX 拡張**: 72以上
- **発生時期**: 2026年3月3〜9日に集中的に発生、3月8日以降も継続

### マルウェアの特徴的な動作

- **言語選別機能**: システムのロケールがロシア語に設定されている場合、実行をスキップ（攻撃者がロシア語圏の可能性を示唆）
- **暗号資産窃取**: 追加ペイロードとして暗号化 JavaScript を通じてウォレット情報を盗取
- **持続性**: 正規のコミット履歴に偽装するため、自動セキュリティスキャンでの検出も困難

### 開発者への影響

- 汚染されたリポジトリから `pip install` を実行、またはコードをクローンして実行するとマルウェアが起動
- 依存関係チェーンを通じた二次感染の可能性
- 個人の認証情報・暗号資産の窃取リスク

### 推奨される対策

1. **GitHub トークンの監査**: 全トークンをローテーション。不審なアクセスログを確認
2. **VS Code / Cursor 拡張の検証**: 信頼性の低い拡張を無効化。インストール済み拡張のリストを確認
3. **コミット履歴の確認**: `git reflog` で強制プッシュの痕跡をチェック
4. **依存関係のピン留め**: `requirements.txt` でバージョンを固定。`pip install` 前にソースを確認
5. **GitHub の署名付きコミット要求**: リポジトリ設定で署名付きコミットを必須に
6. **ブランチ保護の強化**: デフォルトブランチへの強制プッシュを禁止

## 今後の注目点

- 攻撃者の特定とインフラのテイクダウン状況
- GitHub による盗まれたトークンの無効化対応の進捗
- Solana ブロックチェーンを悪用するC2手法が他のマルウェアに波及する可能性
- PyPI / npm のサプライチェーンセキュリティ強化策（署名検証、再現可能ビルド等）
- VS Code / Cursor のエクステンション審査プロセスの見直し
- 開発者コミュニティへの啓発と対策ツールの整備

# Vite 8.0 — Rolldown 統合で 10-30x 高速化

- **日付**: 2026-03-12
- **カテゴリ**: Build Tool
- **ソース**: [Vite 公式ブログ](https://vite.dev/blog/announcing-vite8), [The Register](https://www.theregister.com/2026/03/16/vite_8_rolldown/)

## 概要

Vite 8.0 は Vite 2 以来最大のアーキテクチャ変更を含むメジャーリリースである。これまで開発時に esbuild、本番ビルドに Rollup と2つのバンドラーを使い分けていた構成を、Rust 製の単一バンドラー **Rolldown** に統一した。これにより本番ビルドで 10-30x の高速化を実現し、dev/prod の挙動差異も解消される。

週間ダウンロード数 6,500 万を超えるエコシステムの中核ツールの大規模刷新であり、フロントエンド開発のビルドパイプライン全体に影響を与えるアップデートとなる。

## 主な変更点

### Rolldown 統合

- **単一バンドラー化**: esbuild（dev）+ Rollup（prod）の二重構成を廃止し、Rolldown に一本化
- **Rollup 互換プラグイン API**: 既存の Vite / Rollup プラグインの大半がそのまま動作
- **新機能の解放**: 単一バンドラーにより、フルバンドルモード、柔軟なチャンク分割、モジュールレベルの永続キャッシュ、Module Federation サポートが可能に

### パフォーマンス改善の実績

| プロジェクト | 改善内容 |
|-------------|---------|
| Linear | 本番ビルド: 46s → 6s |
| Beehiiv | ビルド時間 64% 削減 |

### Vite Devtools 標準搭載

デバッグ・分析用の Vite Devtools がビルトインで提供されるようになった。

### @vitejs/plugin-react v6

- Babel への依存を廃止し、**Oxc** を採用
- インストールサイズの大幅縮小
- JSX トランスフォームの高速化

### TypeScript 設定の拡張

- `tsconfig.json` の `paths` 設定の自動解決
- `emitDecoratorMetadata` の組み込みサポート

### インストールサイズ

Vite 7 比で約 15MB 増加:
- lightningcss（約 10MB）がピア依存から標準依存に変更
- Rolldown バイナリ（約 5MB）が追加

## 破壊的変更・移行ガイド

### 必須要件

- **Node.js 20.19 以上** または **22.12 以上**

### 移行の推奨手順

1. **段階的移行**: 大規模プロジェクトでは、まず Vite 7 上で `rolldown-vite` パッケージを試してから Vite 8 にアップグレードすることを推奨
2. **プラグイン互換性の確認**: 大半のプラグインはそのまま動作するが、内部 API に依存しているプラグインは動作確認が必要
3. **公式マイグレーションガイドの確認**: 詳細な変更点と対応方法は公式ドキュメント参照

### 統合ツールチェーン

Vite 8 により、エンドツーエンドのツールチェーンが形成された:
- **ビルドツール**: Vite
- **バンドラー**: Rolldown
- **コンパイラ**: Oxc

3つのチームが密に連携し、スタック全体で一貫した挙動を保証する体制となっている。

## 今後の注目点

- **Rolldown の成熟**: Rolldown 単体でのエコシステム拡大と、Webpack からの移行パスの整備
- **永続キャッシュの安定化**: モジュールレベルのキャッシュによるインクリメンタルビルドのさらなる高速化
- **Module Federation**: マイクロフロントエンド構成での Vite 8 活用事例の増加
- **Vite 9 の方向性**: Rolldown 統合の安定化を経て、さらなる最適化やエッジランタイム対応が見込まれる

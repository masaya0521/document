# Deno — 2.9（deno desktop）

- **日付**: 2026-06-25
- **カテゴリ**: Backend / Runtime
- **ソース**: [Deno 2.9 公式ブログ](https://deno.com/blog/v2.9), [Deno Blog (product-update)](https://deno.com/blog?tag=product-update)

## 概要

Deno 2.9 が 2026-06-25 にリリースされた。目玉は **`deno desktop`** で、すでに使い慣れた Web スタック（HTML/CSS/JS）からネイティブデスクトップアプリケーションを構築できる新機能。Electron 的なボイラープレートを必要とせず、スクリプトや Web フレームワークのプロジェクトを指定するだけで、自己完結した単一バイナリのネイティブアプリを生成する。

加えて、`deno install` が npm・pnpm・yarn・Bun のロックファイルを直接読み込めるようになり、パッケージマネージャの Deno への切り替えが「移行」ではなく「数コマンド」で済むようになった。コールドスタートの高速化やテスト機能の拡充も含まれ、ランタイムとしての成熟が一段と進んでいる。

## 主な変更点

- **deno desktop**: Web スタックからネイティブデスクトップアプリを単一・自己完結バイナリで生成（Electron ボイラープレート不要）
- **パッケージマネージャ移行の簡素化**: npm/pnpm/yarn/Bun のロックファイルを直接読み込み
- **CSS モジュールインポート**: import 属性を使った CSS ファイルのインポートに対応
- **パフォーマンス向上**: コールドスタート時間が約50%削減（34ms → 17ms）、メモリ使用量も削減
- **Node.js 26 互換性**: ネイティブ Node ビルトインの無条件解決に対応
- **テスト機能強化**: スナップショットテスト、パラメータ化テスト（`Deno.test.each`）、リトライ機能
- **deno compile の改善**: `--bundle` フラグでバイナリサイズを大幅削減
- **WebCryptography API 拡張**: 量子耐性アルゴリズム（ML-KEM、ML-DSA、SLH-DSA）を実装

## 破壊的変更・移行ガイド

- 大きな破壊的変更は告知されていない。Node 互換レイヤの解決挙動が変わるため、Node ビルトインに依存するコードは挙動差の確認を推奨
- `deno desktop` は新規機能のため、既存 CLI/サーバ用途への影響はない

## 今後の注目点

- `deno desktop` が Electron / Tauri の代替として実用に耐えるか（配布・自動更新・OS ネイティブ API アクセスの整備状況）
- 量子耐性暗号アルゴリズム対応の Web 標準（WebCrypto）への波及
- Node.js 26 互換の進展と、既存 npm エコシステム資産の移植容易性

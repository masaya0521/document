# npm / Go パッケージ乗っ取りと VS Code タスク悪用の供給網攻撃

- **日付**: 2026-07-01
- **カテゴリ**: Tech / Security
- **ソース**: [The Hacker News](https://thehackernews.com/2026/06/hijacked-npm-and-go-packages-use-vs.html)

## 概要

セキュリティ研究者が、乗っ取られた **2つの npm パッケージと Go パッケージ群**を発見した。これらは Windows / Linux / macOS のホスト上で **Python 製のインフォスティーラー**を展開するように設計されている。問題のパッケージは2026年5月25日に npm へアップロードされ、現在はレジストリから削除済み。

本攻撃の特徴は、実行トリガーに **VS Code の「タスク（tasks）」自動実行**を悪用している点にある。プロジェクトフォルダを VS Code で開いた瞬間にマルウェアが起動するため、開発者がコードを実行する前段階で感染が成立しうる。

## 詳細

### 攻撃メカニズム

感染チェーンは多段構成になっている。

1. **VS Code タスクに実行を隠蔽** — プロジェクトフォルダを開くと自動実行されるタスクとして設定。
2. **ブロックチェーンから暗号化 JS を取得** — マルウェアはブロックチェーンのトランザクションデータから暗号化された JavaScript を取り出す（テイクダウン耐性を狙った C2 配信手法）。
3. **socket.io バックドアを起動** — 攻撃者管理のインフラへ接続。
4. **Python インフォスティーラーを展開** — 最終ペイロード。

### 窃取対象（極めて広範）

このスティーラーは、資格情報・ブラウザ・ウォレット・開発者アーティファクトを幅広く狙う。

- **ブラウザ**: Chromium 系 / Mozilla Firefox に保存されたデータ
- **資格情報管理**: パスワードマネージャー、オーセンティケータ、Windows Credential Manager、Linux Secret Service、KDE Wallet、macOS Keychain
- **暗号資産**: 各種仮想通貨ウォレット
- **開発者情報（特に注意）**: Git 資格情報、GitHub CLI の `hosts.yml`、GitHub Desktop ログ、VS Code のグローバルストレージ
- **クラウドストレージのメタデータ**: Dropbox / Google Drive / OneDrive / iCloud / Box / Mega / pCloud

開発者の認証情報やトークンが根こそぎ抜かれるため、二次的な供給網侵害（盗んだ GitHub 認証で別パッケージを汚染）に発展するリスクが高い。

### 同時期の関連攻撃

2026年6月1日には、**`@redhat-cloud-services` の npm 名前空間で公開されていた少なくとも32パッケージ**が侵害される別の供給網攻撃が発生。攻撃者はコードレビューを完全にバイパスし、**Miasma** と名付けられたペイロードを送り込んだ。npm エコシステムへの攻撃が連続している状況がうかがえる。

## 今後の注目点

- **VS Code タスクの自動実行リスク**: 信頼できないリポジトリを開くだけで任意コードが走りうる設計面の危険性。Workspace Trust の徹底と、`tasks.json` のレビュー習慣が重要。
- **ブロックチェーン経由 C2 の普及**: テイクダウンしにくい配信チャネルとして、トランザクションデータへのペイロード埋め込みが今後増える可能性。
- **開発者を狙う供給網攻撃の常態化**: npm / Go / PyPI など主要レジストリへの連続攻撃。依存関係のピン留め、lockfile 検証、`--ignore-scripts`、SBOM、隔離環境での依存導入などの多層防御が必須に。
- **盗難認証情報の二次利用**: Git / GitHub 認証の窃取は連鎖的な汚染につながるため、トークンの最小権限化と短命化、漏洩検知の整備が問われる。

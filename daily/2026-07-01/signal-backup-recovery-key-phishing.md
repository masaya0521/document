# Signal バックアップ復旧キーを狙うロシア系フィッシング

- **日付**: 2026-07-01
- **カテゴリ**: Tech / Security
- **ソース**: [BleepingComputer](https://www.bleepingcomputer.com/news/security/fbi-russian-hackers-now-target-signal-backup-recovery-keys/) / [The Hacker News](https://thehackernews.com/2026/06/fbi-warns-russian-intelligence-hackers.html)

## 概要

FBI と CISA は、ロシア情報機関（RIS）に紐づく攻撃者が **Signal のバックアップ復旧キー（Backup Recovery Key）** を狙うフィッシングへ手口を進化させていると警告した。これは2026年3月の勧告の更新版で、当初は「認証コードの窃取によるアカウント乗っ取り」が主目的だったが、いまや **復旧キーそのものを奪い、被害者の過去メッセージへアクセスする**方向へシフトしている。

重要なのは、この攻撃が **エンドツーエンド暗号化を破るものではない**点だ。暗号は健在で、狙われているのは人間（ソーシャルエンジニアリング）と、バックアップ機能の運用上の弱点である。

## 詳細

### 攻撃手口

攻撃者は **Signal サポートチームを装い**、以下のような偽のフィッシングメッセージを送る。

> 「イランや旧ソ連諸国のハッカーによる攻撃の急増を受け、Signal は必須の二要素認証を導入します」

そのうえで被害者に対し、

1. バックアップ設定画面を開く
2. 復旧キーをクリップボードにコピーする
3. 「データ消失を防ぐため」と称してそれをメッセージに貼り付ける

よう誘導する。一度復旧キーを渡してしまうと、攻撃者は **そのバックアップを自分のデバイスに復元**し、プライベート・グループを含む過去の会話履歴にアクセスできてしまう。

### 厄介な特性

FBI が特に強調しているのは、**攻撃者が復旧キーを入手した後は、同じ電話番号で新しい Signal アカウントを作り直しても古い盗難キーは無効化されない**という点だ。つまり、被害に気づいてアカウントを作り直しても、過去のバックアップへのアクセスを遮断できない可能性がある。

### 標的と帰属

- **標的**: 諜報価値の高い個人 — 現・元の米国および各国政府高官、軍関係者、政治家、ジャーナリスト、ウクライナの要人。
- **帰属**: ロシア情報機関（RIS）。FSB 国境警備局に組み込まれた工作員や、ロシア軍の委託を受けた実行者を含む。
- **追跡名**: **UNC5792** および **UNC4221** として公的に追跡されている。

## 今後の注目点

- **個人レベルの防御**: 復旧キーは「誰にも・どんなサポートにも渡さない」ことが鉄則。Signal が正規にキーの貼り付けを要求することはない。
- **アプリ側の対策**: 復旧キー流出時にバックアップアクセスを失効させる仕組みの導入が望まれる。プラットフォーム側の運用設計が問われている。
- **手口の横展開**: 同様のソーシャルエンジニアリングが WhatsApp 等の他メッセージングアプリにも波及する可能性。
- **国家関与アクターの継続性**: 地政学的緊張（米・イラン戦争、ウクライナ情勢）を口実にした文面が使われており、当面この種のキャンペーンは継続する見込み。

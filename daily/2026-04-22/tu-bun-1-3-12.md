# Bun v1.3.12 — テスト並列化と install ストリーミングの威力

- **日付**: 2026-04-22
- **カテゴリ**: Backend / Runtime & Tooling
- **ソース**: [Bun v1.3.12 Blog](https://bun.com/blog/bun-v1.3.12), [Bun v1.3.13 Blog](https://bun.com/blog/bun-v1.3.13)

## 概要

2026-04-20 に Bun v1.3.12 がリリースされた。1.3.x 系は「ランタイム + パッケージマネージャ + テストランナー」としての完成度を上げる方向で進行しており、本リリースでは特に **テスト並列実行**と **`bun install` のストリーミング展開** が実務的なインパクトの大きい目玉。1.3.12 では 120 の issue が解消されている。

## 主な変更点

### `bun test` の並列化・分割・差分実行

モノレポや大量テストを抱えるリポジトリ向けに、CI で効く新オプションが揃った。

- `--parallel`: 複数プロセスでテストファイルを分配して実行
- `--isolate`: テストごとにモジュールキャッシュを分離し、グローバル汚染を防ぐ
- `--shard <i/N>`: CI の並列ジョブに合わせてシャーディング（例: `--shard=1/4`）
- `--changed`: git の変更ファイルに関連するテストだけを抽出して実行

```bash
# CI の 4 並列ジョブに分ける例
bun test --parallel --shard=1/4

# ローカルで差分テストだけ素早く回す
bun test --changed
```

### `bun install` のストリーミング展開

従来は `.tgz` をダウンロード → メモリ上で解凍 → 展開という流れだったが、**ダウンロード中の tarball を順次展開する**方式に変わった。

- メモリ消費は **約 17 倍** 削減
- 大量の依存（数百〜数千パッケージ）での install が体感で速くなる
- 不具合時は `BUN_FEATURE_FLAG_DISABLE_STREAMING_INSTALL=1` で旧挙動に戻せる

source map のメモリ使用量も **約 8 倍** 削減された。

### ランタイム・API の強化

- **zlib-ng 採用** により gzip が **5.5 倍** 高速化（`Bun.gunzipSync` や `fetch` のレスポンス解凍）
- `Bun.serve()` が **Range リクエスト** に対応。動画配信や大容量ファイルの部分取得がシンプルに書ける
- `node:crypto` / WebCrypto で **SHA3** をサポート
- Unix ドメインソケット上の WebSocket クライアント `ws+unix://` が利用可能
- ターミナルで `bun ./file.md` すると Markdown を整形表示
- `Bun.WebView` でヘッドレスブラウザ自動化、`Bun.cron()` でプロセス内スケジューラ
- `URLPattern` が **2.3 倍**、`Bun.Glob.scan` が **2 倍** 高速化
- Linux で **cgroup-aware** な並列度決定（コンテナの CPU 制限を考慮）

### 開発者体験

- ネイティブエラーで **async stack trace** が出るようになり、非同期境界の調査が楽になる

## 破壊的変更・移行ガイド

- 明示的な破壊的変更のアナウンスはなく、ほぼ追加 / 改善のリリース。
- ストリーミング install で極めて稀に既存の展開挙動に依存した後処理が壊れる可能性があるが、前述の環境変数でロールバックできる。
- zlib-ng への切り替えで、Bun の gzip 出力バイナリが参照実装（zlib）とビット単位で同一ではなくなるケースがある。ハッシュ値で生成物の同一性を確認しているビルドパイプラインでは挙動を確認する。

## 今後の注目点

- **CI 効率**: `--parallel` + `--shard` + `--changed` の組み合わせは、Jest / Vitest の同等機能に匹敵する CI 設計を Bun 単体で組めることを意味する。Actions のマトリクスジョブとの親和性が高い。
- **メモリ効率**: install のメモリ削減は、Docker ビルドやエッジランタイム（Cloudflare Workers など）での `bun install` 導入のハードルを下げる。
- **Web API 志向**: `URLPattern` の高速化、`Bun.serve()` の Range 対応など、Node 24 と同じ「Web 標準を速く」方向に揃いつつある。Node 24 / Deno / Bun / Workers の相互互換は確実に深まっている。
- **次のリリース（1.3.13）**: 既にブログが公開されており、同じ方向性（テスト並列、install ストリーミング）の継続的改善が見込まれる。

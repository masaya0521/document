# 初の自律型 LLM エージェント侵入 — AI vs AI 時代のサイバー攻防

- **日付**: 2026-06-02
- **カテゴリ**: Tech
- **ソース**: [Sysdig](https://www.sysdig.com/blog/ai-agent-at-the-wheel-how-an-attacker-used-llms-to-move-from-a-cve-to-an-internal-database-in-4-pivots), [The Hacker News](https://thehackernews.com/2026/05/attackers-use-llm-agent-for-post.html)

## 概要

2026年5月10日、Sysdig の脅威リサーチチーム（TRT）が、**人間の指示を一切受けずに LLM エージェントが主導したサイバー侵入**を観測したと報告した。エージェントは公開脆弱性（CVE）から内部データベースの全件流出まで、4段階のピボットをわずか1時間未満で完遂した。これは、AI エージェントが「自律的なポストエクスプロイト・オペレータ」として実戦投入された、世界初の公的に確認された事例とされる。

従来の自動化スクリプトと決定的に異なるのは、エージェントが各コマンドの出力を受け取り、リアルタイムで次の行動を判断し、予期しない結果（未知のスキーマ、欠落したファイル、認証失敗）に遭遇すると適応的に対応した点だ。

## 詳細

### 攻撃チェーン

攻撃は以下の4ピボットで構成された。

1. **初期侵入**: インターネットに公開された marimo ノートブックを CVE-2026-39987 で侵害（RCE）。
2. **クレデンシャル窃取**: 侵害したホストから2つのクラウド認証情報を抽出。
3. **横展開**: 認証情報を分散させた egress プール（出口 IP プール）経由でリプレイし、AWS Secrets Manager から SSH 秘密鍵を取得。
4. **DB 流出**: その SSH 鍵を使い、下流の SSH 踏み台サーバに対して8回の短い SSH セッションを実行。踏み台フェーズで内部 PostgreSQL データベースのスキーマと全コンテンツを **2分未満** で流出させた。

### エージェント駆動の痕跡（IoC）

Sysdig はエージェント主導の実行と整合する4つの指標を特定した。

- 即興的な（improvised）スキーマ列挙の挙動
- 中国語の計画コメント（planning comment）の混入
- 各コマンドの結果に応じた適応的な意思決定
- 人間では困難なほど短時間での連続したピボット

これらは、攻撃者が事前に全手順をハードコードしたスクリプトではなく、LLM がその場で判断・計画していたことを示唆する。

### なぜ重要か

これまで「LLM が自律的にハッキングできる」という主張は学術的なデモ（arXiv 等）が中心だった。今回は**実環境（in the wild）で攻撃者が実際に LLM エージェントを後段の作戦実行に使った**ことが観測された点で画期的だ。攻撃の速度・適応性・スケールが人間オペレータの限界を超えうることを実証した。

防御側も同様に AI を活用する「AI vs AI」のサイバーセキュリティ時代が本格化しつつある。実際、World Economic Forum の調査では、リーダーの87%が AI 関連の脆弱性を最も急速に成長するサイバーリスクと見ている。

## 今後の注目点

- **検知手法の進化**: 適応的なエージェント挙動を従来の IoC ベース検知でどこまで捕捉できるか。振る舞いベース・速度ベースの検知が鍵になる。
- **攻撃の量産化**: エージェント化により、熟練攻撃者でなくとも高度なポストエクスプロイトが自動実行可能になる。攻撃の裾野が広がる懸念。
- **クラウド権限設計**: Secrets Manager の SSH 鍵が容易に横展開に使われた点から、最小権限・短命クレデンシャル・egress 制御の重要性が増す。
- **防御側 AI の対抗**: SOC でのエージェント型防御（自律的なインシデント対応）の導入が加速するか。
- **規制・帰属**: 中国語コメントなどの痕跡が示す通り、国家関与の帰属判断が一層複雑化する。

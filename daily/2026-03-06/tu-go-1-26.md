# Go 1.26 — Green Tea GC と開発者体験の進化

- **日付**: 2026-02-10（リリース日）
- **カテゴリ**: Language
- **ソース**: [公式ブログ](https://go.dev/blog/go1.26), [リリースノート](https://go.dev/doc/go1.26), [InfoWorld](https://www.infoq.com/news/)

## 概要

Go 1.26 は Go 1.25 から 6 ヶ月後にリリースされたメジャーアップデート。最大の目玉は Green Tea ガベージコレクタのデフォルト有効化で、GC オーバーヘッドを 10〜40% 削減する。加えて、`go fix` コマンドの刷新、ゴルーチンリーク検出機能、SIMD パッケージなど、パフォーマンスと開発者体験の両面で大きな進化を遂げている。

## 主な変更点

### Green Tea ガベージコレクタ
- Go 1.25 で実験的に導入され、1.26 でデフォルト有効化
- 小さなオブジェクトのマーキング・スキャンにおけるローカリティと CPU スケーラビリティを改善
- **実測で GC オーバーヘッド 10〜40% 削減**
- 新しい amd64 CPU（Intel Ice Lake / AMD Zen 4 以降）ではベクトル命令を活用し、さらに約 10% の追加改善
- 無効化: `GOEXPERIMENT=nogreenteagc`（1.27 で削除予定）

### ゴルーチンリーク検出（実験的）
- GC のマーキングフェーズを利用し、永久にブロックされたゴルーチンを検出
- 到達不能な同期プリミティブで待機しているゴルーチンを「リーク」として報告
- `GOEXPERIMENT=goroutineleakprofile` で有効化
- `net/http/pprof` エンドポイント `/debug/pprof/goroutineleak` で取得可能
- 使用していない時のランタイムオーバーヘッドはゼロ

### `go fix` の刷新
- 完全に作り直され、コードベースを最新のイディオムやコアライブラリ API に更新するプッシュボタン式ツールに
- モダンな Go 言語機能やライブラリを活用するための数十のフィクサーを内蔵

### 言語変更
- `new` 組み込み関数が式をオペランドとして受け取れるように変更（初期値の指定が可能に）

### 新しい標準ライブラリ
- **`crypto/hpke`**: Hybrid Public Key Encryption（RFC 9180）の実装。ポスト量子ハイブリッド KEM もサポート
- **`simd/archsimd`**（実験的）: アーキテクチャ固有の SIMD 操作へのアクセス。amd64 で 128/256/512 ビットベクトル型をサポート

## 今後の注目点

- Green Tea GC のオプトアウト設定は Go 1.27 で削除予定
- ゴルーチンリーク検出 API のフィードバック期間（プロファイルとして提供するかの検討）
- SIMD パッケージの他アーキテクチャ（arm64 等）への展開
- Go ベース TypeScript コンパイラ（tsgo）のリリースにより、Go エコシステムへの注目度がさらに向上

# FortiBleed: Fortinet 大規模認証情報流出

- **日付**: 2026-06-23
- **カテゴリ**: Tech
- **ソース**: [CISA Advisory](https://www.cisa.gov/news-events/alerts/2026/06/18/cisa-urges-hardening-fortinet-devices-after-reports-credential-exposure) / [The Hacker News](https://thehackernews.com/2026/06/cisa-warns-fortinet-customers-as.html) / [Arctic Wolf](https://arcticwolf.com/resources/blog/active-fortibleed-campaign-impacting-fortinet-devices-across-194-countries/)

## 概要

「FortiBleed」と名付けられたキャンペーンで、ロシア語圏のサイバー犯罪組織が194カ国にわたる **86,644台の Fortinet FortiGate ファイアウォール**から、検証済みの管理者・VPN 認証情報を密かに収集していたことが明らかになった。これはインターネットに露出した Fortinet 境界デバイスの約半数に相当する規模だ。

この事案は2026年6月13日、セキュリティ研究者 Volodymyr "Bob" Diachenko が、検証済み認証情報のデータベースと自動攻撃ツールをホストする攻撃者サーバを偶然発見したことで表面化した。CISA は6月18日に緊急アドバイザリを発出、英 NCSC もグローバル警告を出し、Fortinet PSIRT も公式ブログを公開するなど、わずか6日間で当局・ベンダーが相次いで対応した。

## 詳細

### 攻撃の性質

FortiBleed は単一の新しいゼロデイではなく、**既存脆弱性の悪用・ハッシュクラッキング・認証情報の使い回し・ブルートフォース**を組み合わせたマルチベクトル型の攻撃である。Fortinet の公式見解は「これは新しい Fortinet の脆弱性ではなく、最近のインシデントやアドバイザリとも無関係」というもの。つまり、根本原因は製品の欠陥というより、運用上の弱い認証情報管理にある。

侵害された認証情報の内訳では、汎用 admin アカウントが35%、Fortinet 内蔵システムアカウントが28.3%を占め、合わせて過半数となっている。デフォルト・弱い・使い回された認証情報が攻撃を成立させた構図だ。

### CISA の推奨対応（6/18 アドバイザリ）

- すべてのアクティブな SSL VPN・管理セッションを終了する
- インターネット露出システム上の FortiGate VPN・管理パスワードを即時リセット
- Fortinet 全体に強力なパスワードポリシーを強制
- 全管理者・VPN ユーザーアカウントに MFA を実装
- 最新の FortiOS 7.4 / 7.6 / 8.0 ブランチへアップグレード
- ログを精査し不正ログイン・設定変更を確認
- 管理インターフェースへの外部アクセスを制限・除去
- 侵害後の不審な挙動が見られるデバイスは交換を検討

## 今後の注目点

- **二次被害の連鎖**: 窃取された VPN/admin 認証情報を起点としたランサムウェア・横展開のリスク。攻撃者サーバには自動攻撃ツールが併設されていた。
- **「脆弱性ではない」問題の難しさ**: パッチでは解決せず、各組織の認証情報衛生（クレデンシャル・ハイジーン）が問われる。MFA 未導入組織が特に危険。
- **境界デバイスへの依存リスク**: VPN/ファイアウォールという「守りの要」が単一障害点になる構造的課題が改めて浮き彫りに。
- **継続的なデータベース拡大**: 侵害台数は6/13公表時から6/19の86,644台へと増加しており、今後さらに拡大する可能性。
